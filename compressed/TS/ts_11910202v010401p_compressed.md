# ETSI TS 119 102-2 V1.4.1: Procedures for Creation and Validation of AdES Digital Signatures; Part 2: Signature Validation Report
**Source**: ETSI | **Version**: V1.4.1 | **Date**: 2023-06 | **Type**: Normative (Technical Specification)
**Original**: RTS/ESI-0019102-2v1.4.1 (https://forge.etsi.org/rep/esi/x19_10202_validation_report)

## Scope (Summary)
Specifies a general structure and an XML format for reporting the validation of AdES digital signatures (CAdES, XAdES, PAdES, JAdES) as defined in ETSI EN 319 122-1, EN 319 132-1, EN 319 142-1, TS 119 182-1. The present document is aligned with the requirements of ETSI EN 319 102‑1.

## Definitions and Abbreviations
- **AdES (digital) signature**: digital signature that is either a CAdES, PAdES, or XAdES signature.
- **advanced electronic signature**: as defined in Regulation (EU) No 910/2014.
- **CAdES signature**: digital signature satisfying ETSI EN 319 122-1 or EN 319 122-2.
- **Coordinated Universal Time (UTC)**: as defined in ITU-R TF.460-6.
- **data object**: actual binary/octet data being operated on (transformed, digested, or signed).
- **data to be signed formatted (DTBSF)**: data created from data objects by formatting and sequencing for computation of DTBSR.
- **data to be signed representation (DTBSR)**: hash of DTBSF used to compute the digital signature value.
- **JAdES signature**: digital signature satisfying ETSI TS 119 182-1.
- **PAdES signature**: digital signature satisfying ETSI EN 319 142-1 or EN 319 142-2.
- **public key**: that key of an entity's key pair which is publicly known.
- **qualified electronic signature**: as defined in Regulation (EU) No 910/2014.
- **signature validation report**: see ETSI EN 319 102‑1, clause 5.1.3.
- **(electronic) time-stamp**: data in electronic form which binds other electronic data to a particular time establishing evidence that these data existed at that time.
- **trusted list**: list providing status and history of trust services from TSPs regarding compliance with applicable requirements and legislation.
- **XAdES signature**: digital signature satisfying ETSI EN 319 132-1 or EN 319 132-2.

**Abbreviations**: AdES, CRL, DA, DER, DSS, DTBSF, DTBSR, OCSP, OID, PDF, POE, SD, SDR, SVA, TSP, TU, URI, URL, URN, UTC, VRI, W3C, XML.

## 4 Signature Validation Report Structure

### 4.1 General Provisions

#### 4.1.1 Report Structure
The **Signature Validation Report** shall consist of:
- One or more **Signature Validation Report Elements** (clause 4.3) containing overall validation status and additional information per signature.
It may also contain:
- **Signature Validation Objects Element** (clause 4.4) containing objects used during validation (trust anchors, CRLs, time-stamps).
- **Validator Information Element** (clause 4.5) identifying the validating entity.
- **Validation Report Signature Element** (clause 4.6) containing a signature over the report.

**XML namespace prefixes:**
- `vr:` (or no prefix) = namespace for this document: `http://uri.etsi.org/19102/v1.4.1#`
- `ds:` = W3C XML Signature namespace (http://www.w3.org/2000/09/xmldsig#)
- `XAdES:` = namespace for XAdES (http://uri.etsi.org/01903/v1.3.2)
- `tsl:` = namespace for Trusted Lists (http://uri.etsi.org/02231/v2#)
- `xs:` = XML Schema namespace (http://www.w3.org/2001/XMLSchema)

In case of discrepancies between XML schema excerpts in the document and the actual schema files, the XML Schema files shall take precedence.

##### 4.1.1.2 Validation Object Reference Element
Used to reference another element within the report (e.g., signature validation objects or Signature Identification Element).
- **XML**: type `VOReferenceType`, attribute `VOReference` (xs:IDREFS, required).

##### 4.1.1.3 Typed Data Type
A generic Type-Value tuple.
- **XML**: type `TypedDataType` with `Type` (xs:anyURI) and `Value` (xs:anyType).

##### 4.1.1.4 Any Type
Allows inclusion of optional arbitrary elements.
- **XML**: type `AnyType` (mixed).

##### 4.1.1.5 Signature Reference
References a specific electronic signature.
- **XML**: type `SignatureReferenceType`, choice of: `CanonicalizationMethod`, `DigestMethod`, `DigestValue`; or `XAdESSignaturePtr`; or `PAdESFieldName`; or `OtherInformation`.

### 4.2 Validation Report Element
- **Presence**: Mandatory.
- **Description**: Wrapper for reports on one or more signatures.
- **Content**: Shall contain one or more `SignatureValidationReport` elements (clause 4.3). May contain `SignatureValidationObjects`, `SignatureValidator`, and `<ds:Signature>`.
- **XML**: element name `ValidationReport`, type `ValidationReportType`.

### 4.3 Signature Validation Report Element
- **Presence**: Mandatory.
- **Description**: Represents validation information for a single signature.
- **Content**: Shall contain sequence of elements from clauses 4.3.3 to 4.3.12.
- **XML**: type `SignatureValidationReportType`.

#### 4.3.3 Signature Identification Element
- **Presence**: Conditional – shall be present unless signature validation status indicates TOTAL-FAILED with FORMAT_FAILURE (signature unprocessable). May be present for validation objects.
- **Content**: Shall contain DTBSR (DigestAlgAndValue), hash algorithm identifier, indication whether DTBSF/DTBSR processed, and whether SD/SDR processed. May contain unique identifier, digital signature value, DA identifier, other identifying elements.
- **XML**: type `SignatureIdentifierType`, element `DigestAlgAndValue`, booleans `HashOnly` and `DocHashOnly`, optional `ds:SignatureValue`, `DAIdentifier`, `OtherInformation`, attribute `id`.

#### 4.3.4 Signature Validation Status Indication
- **Presence**: Mandatory.
- **Content**: One `MainIndication` and optionally one or more `SubIndication` elements.
- **Main URIs** (for signature validation): TOTAL-PASSED (`urn:etsi:019102:mainindication:total-passed`), TOTAL-FAILED, INDETERMINATE. For individual constraint or validation object: PASSED, FAILED, INDETERMINATE.
- **SubIndication**: URIs as per Table 2 (e.g., FORMAT_FAILURE, HASH_FAILURE, SIG_CRYPTO_FAILURE, REVOKED, etc.), including CUSTOM and REVOCATION_OUT_OF_BOUNDS_NO_POE.
- **XML**: type `ValidationStatusType` with `MainIndication` (xs:anyURI), `SubIndication` (xs:anyURI, 0..unbounded), `AssociatedValidationReportData` (0..unbounded).

#### 4.3.5 Validation Constraints Evaluation Report
- **Presence**: Conditional – shall be present in signature validation report; may be present for validation objects.
- **Content**: When a formal signature validation policy (per TS 119 172-1) is selected, shall contain `SignatureValidationPolicy` element. Shall contain individual `ValidationConstraint` elements for constraints applied, disabled, or overridden.
- **XML**: type `ValidationConstraintsEvaluationReportType` with optional `SignatureValidationPolicy` and 0..unbounded `ValidationConstraint`.

##### 4.3.5.3 Formal Policy Element
- **Presence**: Conditional – shall be present when formal policy selected.
- **Content**: Policy identifier (unique), optionally policy name, formal policy URI, human-readable policy URI, reference to object in Signature Validation Objects.
- **XML**: type `SignatureValidationPolicyType` with `SignaturePolicyIdentifier` (XAdES type), `PolicyName`, `FormalPolicyURI`, `ReadablePolicyURI`, `FormalPolicyObject`.

##### 4.3.5.4 Individual Validation Constraint Report Element
- **Presence**: Optional.
- **Content**: Validation constraint identifier (URI), constraint status (applied/disabled/overridden), validation result (main indication + optional sub-indication), optional parameters and indications for indeterminate.
- **XML**: type `IndividualValidationConstraintReportType`. Constraint status: element `ConstraintStatus` of type `ConstraintStatusType` with `Status` and optional `OverriddenBy`. Status URIs: `urn:etsi:019102:constraintStatus:applied`, `:disabled`, `:overridden`.

#### 4.3.6 Signature Validation Time Info
- **Presence**: Conditional – shall be present in signature validation report; may be for validation objects.
- **Content**: Date/time validation performed; date/time of POE of signature (BestSignatureTime). May include POE source and reference to object.
- **XML**: type `ValidationTimeInfoType` with `ValidationTime` (xs:dateTime) and `BestSignatureTime` (POEType).

#### 4.3.7 Signer's Document Element
- **Presence**: Conditional – shall be present in signature validation report; may be for validation objects.
- **Content**: Shall contain either SDR (as DigestAlgAndValue) or reference to validation object containing SDR, or both. May contain reference to SD.
- **XML**: type `SignersDocumentType`. Choice: `DigestAlgAndValue` + optional `SignersDocumentRepresentation` (VOReference), or just `SignersDocumentRepresentation`. Optional `SignersDocumentRef`.

#### 4.3.8 Signature Attribute Element
- **Presence**: Conditional – shall be present whenever signature contains attributes.
- **Content**: One instance per attribute, each containing type, whether signed/unsigned, attribute-dependent info (Annex A), and optional references to validation objects.
- **XML**: type `SignatureAttributesType` with choice of elements defined in Annex A. Each attribute element extends `AttributeBaseType` with boolean `Signed`.

#### 4.3.9 Signer Information Element
- **Presence**: Conditional – shall be present when signing certificate identified.
- **Content**: Reference to signing certificate object (VOReference). May include human-readable signer representation. When pseudonym used, shall indicate.
- **XML**: type `SignerInformationType` with `SignerCertificate`, optional `Signer`, `OtherInformation`, attribute `Pseudonym`.

#### 4.3.10 Signature Quality Element
- **Presence**: Optional.
- **Content**: One or more URIs indicating signature quality (e.g., qualified electronic signature).
- **XML**: type `SignatureQualityType` with 1..unbounded `SignatureQualityInformation` (xs:anyURI).

#### 4.3.11 Signature Validation Process Information Element
- **Presence**: Optional.
- **Content**: Validation process ID URI (Basic, LTVM, LTA) or other URI, service policy URI, practice statement URI, other info.
- **XML**: type `SignatureValidationProcessType` with optional `SignatureValidationProcessID`, `SignatureValidationServicePolicy`, `SignatureValidationPracticeStatement`, `OtherInformation`.

#### 4.3.12 Associated Validation Report Data Element
- **Presence**: Optional.
- **Content**: May contain: `TrustAnchor` (VOReference), `CertificateChain`, `RelatedValidationObject` (VOReference), `RevocationStatusInformation`, `CryptoInformation`, `AdditionalValidationReportData`.
- **XML**: type `ValidationReportDataType`.

##### 4.3.12.3 Trust Anchor Element
- **Presence**: Optional.
- **Content**: Reference to object (certificate or public key) used as trust anchor.
- **XML**: element `TrustAnchor` of type `VOReferenceType`.

##### 4.3.12.4 Certificate Chain Element
- **Presence**: Optional.
- **Content**: List of references: signing certificate first, trust anchor last, intermediate certificates in between.
- **XML**: type `CertificateChainType` with `SigningCertificate`, 0..unbounded `IntermediateCertificate`, optional `TrustAnchor`, optional `OtherInformation`.

##### 4.3.12.5 Related Data Objects Element
- **Presence**: Optional.
- **Content**: References to objects causing a sub-indication (e.g., timestamps in TIMESTAMP_ORDER_FAILURE).
- **XML**: type `VOReferenceType`.

##### 4.3.12.6 Revocation Status Information Element
- **Presence**: Conditional – shall be present when a certificate is revoked.
- **Content**: Reference to certificate, revocation time (UTC), reason (URI), optional revocation object (CRL/OCSP), other info.
- **XML**: type `RevocationStatusInformationType`. RevocationReason URIs: `unspecified`, `keyCompromise`, `cACompromise`, `affiliationChanged`, `superseded`, `cessationOfOperation`, `certificateHold`, `privilegeWithdrawn`, `removeFromCRL`, `aACompromise`.

##### 4.3.12.7 Crypto Information Element
- **Presence**: Conditional – shall be present when main indication INDETERMINATE with sub-indication CRYPTO_CONSTRAINTS_FAILURE. May be present otherwise.
- **Content**: Reference to object/signature, algorithm URI, whether algorithm is considered secure, optional parameters, expiry time.
- **XML**: type `CryptoInformationType` with `ValidationObjectId`, `Algorithm`, optional `AlgorithmParameters`, `SecureAlgorithm`, optional `NotAfter`, `OtherInformation`.

##### 4.3.12.8 Additional Validation Report Data
- **Presence**: Optional.
- **Content**: Any additional info (Type-Value tuples).
- **XML**: type `AdditionalValidationReportDataType` with 1..unbounded `ReportData` (TypedDataType).

### 4.4 Signature Validation Objects
- **Presence**: Optional.
- **Description**: Container for validation objects used during validation (avoids duplication).
- **Content**: Sequence of `ValidationObject` elements. Each has: unique id, object type URI, object representation, optional POE, POEProvisioning, ValidationReport.
- **XML**: type `ValidationObjectListType`, each element `ValidationObject` of type `ValidationObjectType`. ObjectType URIs: `certificate`, `CRL`, `OCSPResponse`, `timestamp`, `evidencerecord`, `publicKey`, `signedData`, `other`.

#### 4.4.3 Object Identifier
- **Presence**: Mandatory.
- **XML**: attribute `id` (xs:ID), unique within report.

#### 4.4.4 Object Type
- **Presence**: Mandatory.
- **XML**: element `ObjectType` (xs:anyURI) using URIs from 4.4.2.

#### 4.4.5 Validation Object
- **Presence**: Mandatory.
- **Content**: One or more of: direct content (`anyType`), base64, digest (DigestAlgAndValue), URI.
- **XML**: type `ValidationObjectRepresentationType`.

#### 4.4.6 Proof Of Existence (POE)
- **Presence**: Optional.
- **Content**: POE time (UTC), type of proof (derived, provided, policy), optional reference to object essential for proof.
- **XML**: type `POEType`. TypeOfProof URIs: `urn:etsi:019102:poetype:validation`, `:provided`, `:policy`.

#### 4.4.7 POE Provisioning
- **Presence**: Optional.
- **Content**: POE time, list of references to signatures or validation objects covered by that proof.
- **XML**: type `POEProvisioningType` with `POETime`, 0..unbounded `ValidationObject` (VOReference), 0..unbounded `SignatureReference`.

#### 4.4.8 Validation Object Validation Report
- **Presence**: Optional (for signed objects).
- **Content**: Validation report conforming to this document. The main report’s signature protects this sub-report.

### 4.5 Validator Information
- **Presence**: Optional.
- **Content**: Digital identity of validation service (per ETSI TS 119 612 clause 5.5.3), optionally TSP information (clause 5.4), other info.
- **XML**: type `SignatureValidatorType` with 1..unbounded `DigitalId` (tsl:DigitalIdentityType), optional `TSPInformation` (tsl:TSPInformationType), optional `OtherInformation`.

### 4.6 Validation Report Signature
- **Presence**: Optional.
- **Content**: Signature over the report by the validation service. Example: enveloped signature over entire report.

## Informative Annexes (Condensed)

### Annex A (Normative): Signature Attribute Representation
Defines XML representation of signed and unsigned attributes from CAdES, XAdES, PAdES, JAdES. Each attribute element extends `AttributeBaseType` and uses `Signed` attribute. The `SignatureAttributesType` contains a choice of elements corresponding to each defined attribute.

- **A.1 SignatureAttributesType**: Choice of elements for all standard attributes. Extension point via `OtherInformation` (AnyType). Convention: references to actual XAdES properties prefixed with `XAdES:`; children of `SignatureAttributes` use unqualified names.
- **A.2 SigningTime**: Reported via `SigningTime` element of type `SASigningTimeType` with `Time` (xs:dateTime). Mapping: CAdES uses signing-time attribute; XAdES uses `XAdES:SigningTime`; PAdES uses M entry in Signature dictionary (clause 5) or same as XAdES (clause 6); JAdES uses `sigT` header.
- **A.3 SigningCertificate**: Reported via `SigningCertificate` element of type `SACertIDListType` containing 0..unbounded `CertID` (SACertIDType). Each `CertID` includes optional `X509IssuerSerial` (base64 DER-encoded IssuerSerial), `ds:DigestMethod`, `ds:DigestValue`. References to certificates not present as validation objects use `CertID`; otherwise use `AttributeObject`.
- **A.4 DataObjectFormat**: `DataObjectFormat` element of type `SADataObjectFormatType` with optional `ContentType` (xs:anyURI) and `MimeType` (xs:string). Mappings per format.
- **A.5 CommitmentTypeIndication**: `CommitmentTypeIndication` element of type `SACommitmentTypeIndicationType` with `CommitmentTypeIdentifier` (xs:anyURI).
- **A.6 AllDataObjectsTimeStamp**: `AllDataObjectsTimeStamp` of type `SATimestampType` with `TimeStampValue` (xs:dateTime) and `AttributeObject` (VOReference) for the time-stamp token.
- **A.7 IndividualDataObjectsTimeStamp**: Same type.
- **A.8 SigPolicyIdentifier**: `SigPolicyIdentifier` of type `SASigPolicyIdentifierType` with `SigPolicyId` (xs:anyURI).
- **A.9 SignatureProductionPlace**: `SignatureProductionPlace` of type `SASignatureProductionPlaceType` with one or more `AddressString` entries.
- **A.10 SignerRole**: `SignerRole` of type `SASignerRoleType` with `RoleDetails` (SAOneSignerRoleType) containing `Role` (xs:string) and `EndorsementType` (certified/claimed/signed).
- **A.11 CounterSignature**: `CounterSignature` of type `SACounterSignatureType` with `CounterSignature` as `SignatureReferenceType`.
- **A.12 SignatureTimeStamp**: Same type as A.6.
- **A.13 CompleteCertificateRefs**: `CompleteCertificateRefs` of type `SACertIDListType`.
- **A.14 CompleteRevocationRefs**: `CompleteRevocationRefs` of type `SARevIDListType` containing choice of `CRLID` (SACRLIDType with digest) or `OCSPID` (SAOCSPIDType with ProducedAt, ResponderIDByName/ByKey).
- **A.15 AttributeCertificateRefs**: Same as A.13.
- **A.16 AttributeRevocationRefs**: Same as A.14.
- **A.17 SigAndRefsTimeStamp**: Same as A.6.
- **A.18 RefsOnlyTimeStamp**: Same as A.6.
- **A.19 CertificateValues**: `CertificateValues` of type `AttributeBaseType`; each certificate referenced via `AttributeObject`.
- **A.20 RevocationValues**: Similar, referencing CRL/OCSP/other objects.
- **A.21 AttrAuthoritiesCertValues**: Same pattern.
- **A.22 AttributeRevocationValues**: Same pattern.
- **A.23 TimeStampValidationData**: `TimeStampValidationData` of type `AttributeBaseType`; certificates and revocation values as in A.19/A.20.
- **A.24 ArchiveTimeStamp**: `ArchiveTimeStamp` of type `SATimestampType`; multiple per CAdES archive-time-stamp attributes.
- **A.25 RenewedDigests**: `RenewedDigests` of type `SAListOfIntegersType` (list of integers identifying signed references order).
- **A.26 MessageDigest**: `MessageDigest` of type `SAMessageDigestType` with `Digest` (xs:base64Binary).
- **A.27 DSS**: `DSS` of type `SADSSType` with `Certs`, `CRLs`, `OCSPs` (each VOReferenceType) for PAdES.
- **A.28 VRI**: `VRI` of type `SAVRIType` with `Certs`, `CRLs`, `OCSPs`, `TU` (xs:string), `TS` (SATimestampType) for PAdES.
- **A.29 DocTimeStamp**: `DocTimeStamp` of type `SATimestampType` for PAdES document time-stamps.
- **A.30 Reason**: `Reason` of type `SAReasonType` with `ReasonElement` (xs:string) for PAdES Signature dictionary.
- **A.31 Name**: `Name` of type `SANameType` with `NameElement`.
- **A.32 ContactInfo**: `ContactInfo` of type `SAContactInfoType` with `ContactInfoElement`.
- **A.33 SubFilter**: `SubFilter` of type `SASubFilterType` with `SubFilterElement`.
- **A.34 ByteRange**: `ByteRange` of type `SAListOfIntegersType` (even number of integers, pairs of starting offset and length).
- **A.35 Filter**: `Filter` of type `SAFilterType` with `Filter` (xs:string).
