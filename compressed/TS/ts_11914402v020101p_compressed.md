# ETSI TS 119 144-2: Electronic Signatures and Infrastructures (ESI); PAdES digital signatures - Testing Conformance and Interoperability; Part 2: Test suites for testing interoperability of PAdES baseline signatures
**Source**: ETSI | **Version**: V2.1.1 | **Date**: 2016-06 | **Type**: Normative  
**Original**: ETSI TS 119 144-2 (2016-06), RTS/ESI-0019144-2

## Scope (Summary)
This document defines test suites to assess interoperability between implementations claiming conformance to PAdES baseline signatures [2] at four levels (B-B, B-T, B-LT, B-LTA), plus augmentation and negative test cases. Test suites are agnostic of PKI infrastructure.

## Normative References
- [1] ETSI EN 319 122-1: "CAdES digital signatures; Part 1: Building blocks and CAdES baseline signatures"
- [2] ETSI EN 319 142-1: "PAdES digital signatures; Part 1: Building blocks and PAdES baseline signatures"

## Definitions and Abbreviations
- **Negative test case**: Test case for a signature whose validation according to ETSI EN 319 102-1 [i.3] would not result in TOTAL-PASSED.
- **Abbreviations**: Per ETSI TR 119 001 [i.2].

## Overview
Test suites correspond to PAdES baseline levels defined in ETSI EN 319 142-1 [2]:
- B-B, B-T, B-LT, B-LTA interoperability tests
- Augmentation from B-T to B-LTA
- Negative test cases for B-B, B-T, and B-LTA.

## 5 Testing interoperability of PAdES-B-B signatures
Mandatory attributes per [2] clauses 6.2/6.3 **shall** be present.

| TC ID | Description | Pass Criteria | Required Signature Attributes |
|-------|-------------|---------------|-------------------------------|
| PAdES/BB/1 | Simplest PAdES‑B‑B with only mandatory attributes. | Positive validation. Signature dictionary **shall** contain Type, Contents, Filter, SubFilter, M, ByteRange. CMS binary **shall** include SigningCertificate, ContentType, ESSSigningCertificateV2, MessageDigest. | SignatureDictionary: Type (Sig), Filter (Adobe.PPKLite), SubFilter (ETSI.CAdES.detached), M, ByteRange, Contents (DER CMS with Certificates (SigningCertificate), ContentType, MessageDigest, ESSSigningCertificateV2) |
| PAdES/BB/2 | Adds Location, ContactInfo, Reason entries in signature dictionary. | Positive validation. Signature dictionary **shall** include Type, Contents, Filter, SubFilter, M, Location, ContactInfo, Reason, ByteRange. CMS attributes per BB/1. | As BB/1 plus Location, ContactInfo, Reason in SignatureDictionary. |
| PAdES/BB/3 | Adds ContentTimeStamp attribute in CMS. | Positive validation. CMS **shall** include ContentTimeStamp in addition to BB/1 attributes. | SignatureDictionary per BB/1. CMS adds ContentTimeStamp. |
| PAdES/BB/4 | Includes ClaimedAttribute in SignerAttributesV2. | Positive validation. CMS **shall** include ClaimedAttribute (under SignerAttributesV2) in addition to BB/1 attributes. | SignatureDictionary per BB/1. CMS adds SignerAttributesV2 (ClaimedAttribute). |
| PAdES/BB/5 | Includes CertifiedAttributeV2 in SignerAttributesV2. | Positive validation. CMS **shall** include CertifiedAttributesV2 (under SignerAttributesV2) in addition to BB/1 attributes. | SignatureDictionary per BB/1. CMS adds SignerAttributesV2 (CertifiedAttributeV2). |
| PAdES/BB/6 | Includes SignaturePolicyIdentifier attribute. Signature dictionary includes Reason, Location. | Positive validation. CMS **shall** include SignaturePolicyIdentifier. | SignatureDictionary: Type, Contents, Filter, SubFilter, M, Reason, Location, ByteRange. CMS adds SignaturePolicyIdentifier. |
| PAdES/BB/7 | Uses SHA‑1 digest algorithm. Only mandatory attributes. | Positive validation. CMS **shall** include ESSSigningCertificate (note: not V2). | SignatureDictionary per BB/1. CMS includes ESSSigningCertificate (instead of V2). |

## 6 Testing interoperability of PAdES-B-T signatures
Shall be built on B‑B conformant signatures. A B‑T signature **shall** include a timestamp token (SignatureTimeStamp). Mandatory attributes per [2] clauses 6.2/6.3 **shall** be present.

| TC ID | Description | Pass Criteria | Required Signature Attributes |
|-------|-------------|---------------|-------------------------------|
| PAdES/BT/1 | Simplest PAdES‑B‑T with M entry in signature dictionary and SignatureTimeStamp. | Positive validation. Signature dictionary **shall** contain Type, Contents, Filter, SubFilter, M, ByteRange. CMS **shall** include ContentType, ESSSigningCertificateV2, MessageDigest (signed) and SignatureTimeStamp (unsigned). | SignatureDictionary per BB/1. CMS adds SignatureTimeStamp (unsigned). |
| PAdES/BT/2 | Includes ClaimedAttribute. | Positive validation. CMS **shall** include ClaimedAttribute and SignatureTimeStamp. | SignatureDictionary per BT/1. CMS adds ClaimedAttribute and SignatureTimeStamp. |
| PAdES/BT/3 | Includes SignaturePolicyIdentifier and CommitmentTypeIndication. | Positive validation. CMS **shall** include SignaturePolicyIdentifier, CommitmentTypeIndication, and SignatureTimeStamp. | SignatureDictionary per BT/1. CMS adds SignaturePolicyIdentifier, CommitmentTypeIndication, SignatureTimeStamp. |
| PAdES/BT/4 | Includes ContentTimeStamp (of data before signing) plus SignatureTimeStamp. | Positive validation. CMS **shall** include ContentTimeStamp (signed) and SignatureTimeStamp (unsigned). | SignatureDictionary per BT/1. CMS adds ContentTimeStamp, SignatureTimeStamp. |

## 7 Testing interoperability of PAdES-B-LT signatures
Shall be built on B‑T conformant signatures. A B‑LT signature **shall** include DSS dictionary with certificate values and status. Mandatory attributes per [2] clauses 6.2/6.3 **shall** be present.

| TC ID | Description | Pass Criteria | Required Signature Attributes |
|-------|-------------|---------------|-------------------------------|
| PAdES/BLT/1 | Simplest PAdES‑B‑LT with DSS containing Certs and CRLs. | Positive validation. Signature dictionary **shall** contain Type, Contents, Filter, SubFilter, M, ByteRange. CMS includes ContentType, ESSSigningCertificateV2, MessageDigest, SignatureTimeStamp. DSS **shall** contain Type, Certs, CRLs. | SignatureDictionary per BT/1. CMS per BT/1. DSS: Type, Certs, CRLs. |
| PAdES/BLT/2 | DSS contains Certs and OCSPs. | Positive validation. DSS **shall** contain Type, Certs, OCSPs. | As BLT/1 but DSS: Type, Certs, OCSPs. |
| PAdES/BLT/3 | DSS contains Certs, CRLs, VRI; VRI contains Cert, CRL. | Positive validation. DSS **shall** contain Type, Certs, CRLs, VRI. VRI **shall** contain Type, Cert, CRL. | As BLT/1 plus DSS VRI with Type, Cert, CRL. |
| PAdES/BLT/4 | DSS contains Certs, OCSPs, VRI; VRI contains Cert, OCSP. | Positive validation. DSS **shall** contain Type, Certs, OCSPs, VRI. VRI **shall** contain Type, Cert, OCSP. | As BLT/2 plus DSS VRI with Type, Cert, OCSP. |

## 8 Testing interoperability of PAdES-B-LTA signatures
Shall be built on B‑LT conformant signatures. A B‑LTA signature **shall** include one or more DTS dictionaries. Mandatory attributes per [2] clauses 6.2/6.3 **shall** be present.

| TC ID | Description | Pass Criteria | Required Signature Attributes |
|-------|-------------|---------------|-------------------------------|
| PAdES/BLTA/1 | Simplest PAdES‑B‑LTA with DSS (Certs, CRLs) and one DTS. | Positive validation. Signature dictionary per BLT/1. CMS per BLT/1. DSS **shall** contain Type, Certs, CRLs. DTS **shall** contain Type, SubFilter, Contents. | SignatureDictionary per BT/1. CMS per BT/1. DSS: Type, Certs, CRLs. DTS: Type, SubFilter, Contents. |
| PAdES/BLTA/2 | DSS contains Certs and OCSPs; one DTS. | As BLTA/1 but DSS contains Certs, OCSPs. | Same as BLTA/1 but DSS: Type, Certs, OCSPs. |
| PAdES/BLTA/3 | DSS contains Certs, CRLs, VRI; VRI contains Cert, CRL; one DTS. | DSS **shall** contain Type, Certs, CRLs, VRI; VRI **shall** contain Type, Cert, CRL. DTS per above. | As BLT/3 plus DTS. |
| PAdES/BLTA/4 | DSS contains Certs, OCSPs, VRI; VRI contains Cert, OCSP; one DTS. | DSS **shall** contain Type, Certs, OCSPs, VRI; VRI **shall** contain Type, Cert, OCSP. DTS per above. | As BLT/4 plus DTS. |

## 9 Testing PAdES baseline signatures augmentation interoperability
Test cases augment B‑T signatures to B‑LTA by adding DSS (with certificates/status) and DTS.

| TC ID | Description | Pass Criteria | Required Attributes |
|-------|-------------|---------------|---------------------|
| PAdES/Aug/1 | Input: B‑T signature. Validate, add validation material (CRLs) to DSS, then add one DTS. | Signature **shall** become B‑LTA with DSS (Type, Certs, CRLs) and DTS (Type, SubFilter, Contents) added to original B‑T signature. | Original B‑T per BT/1. Added DSS: Type, Certs, CRLs. Added DTS: Type, SubFilter, Contents. |
| PAdES/Aug/2 | Input: B‑T signature. Use OCSPs instead of CRLs. | As Aug/1 but DSS contains Certs, OCSPs. | Same but DSS: Type, Certs, OCSPs. |
| PAdES/Aug/3 | Input: B‑LTA signature (with one DTS). Validate, add validation material (CRLs) for timestamp certificate, then add another DTS. | Signature **shall** contain original B‑LTA plus new DSS (for timestamp cert) and new DTS. | Original B‑LTA per BLTA/1 or similar. New DSS: Type, Certs, CRLs. New DTS: Type, SubFilter, Contents. |
| PAdES/Aug/4 | Input: B‑LTA signature. Use OCSPs for timestamp cert. | As Aug/3 but new DSS contains OCSPs. | Same but new DSS: Type, Certs, OCSPs. |

## 10 Testing negative PAdES baseline signatures

### 10.1 PAdES-B-B signatures test cases
| TC ID | Description |
|-------|-------------|
| PAdES/BBN/1 | Verify a B‑B signature with wrong signature (hash mismatch). |
| PAdES/BBN/2 | Verify a B‑B signature with untrusted signing certificate. |
| PAdES/BBN/3 | Verify a B‑B signature with expired signing certificate. |
| PAdES/BBN/4 | Verify a B‑B signature with revoked/suspended signing certificate. |
| PAdES/BBN/5 | Verify a B‑B signature with CA certificate revoked/suspended. |
| PAdES/BBN/6 | Verify a B‑B signature with wrong byte range. |

### 10.2 PAdES-B-T signatures test cases
| TC ID | Description |
|-------|-------------|
| PAdES/BTN/1 | Verify a B‑T signature where signer certificate expired before SignatureTimeStamp. |
| PAdES/BTN/2 | Verify a B‑T signature where signer certificate revoked before SignatureTimeStamp. |
| PAdES/BTN/3 | Verify a B‑T signature where hash in SignatureTimeStamp does not match signature value. |
| PAdES/BTN/4 | Verify a B‑T signature where timestamp signer certificate revoked. |
| PAdES/BTN/5 | Verify a B‑T signature where timestamp signer certificate expired. |
| PAdES/BTN/6 | Verify a B‑T signature where timestamp signer certificate from untrusted CA. |
| PAdES/BTN/7 | Verify a B‑T signature where timestamp signer CA certificate revoked/suspended. |

### 10.3 PAdES-B-LTA signatures test cases
| TC ID | Description |
|-------|-------------|
| PAdES/BLTAN/1 | Verify a B‑LTA signature where SignatureTimeStamp is later than Document Time Stamp. |
| PAdES/BLTAN/2 | Verify a B‑LTA signature with wrong signature in Document Time Stamp. |
