# ETSI TS 102 778-1: PDF Advanced Electronic Signature Profiles; Part 1: PAdES Overview - a framework document for PAdES
**Source**: ETSI | **Version**: V1.1.1 | **Date**: July 2009 | **Type**: Informative
**Original**: ETSI TS 102 778-1 (2009-07)

## Scope (Summary)
Provides a framework for the set of PDF Advanced Electronic Signature (PAdES) profiles specified in the multi-part deliverable. Describes general PDF signature support, lists features of each profile, and explains how profiles may be combined. This document is for information only; normative requirements are in other parts.

## Definitions and Abbreviations
### Definitions
- **certification signature**: signature used with modification detection permissions (MDP) as defined by ISO 32000-1 [1], clause 12.8.2.2
- **conforming signature handler**: software that performs digital signature operations (signing/verifying) in conformance with ISO 32000-1 [1] and the appropriate profile
- **PDF serial signature**: workflow where subsequent signers sign the document plus previous signatures and modifications (e.g., form fill-in)
- **PDF signature**: binary data object based on CMS (RFC 3852 [5]) or related syntax, placed within a PDF document structure as per ISO 32000-1 [1], clause 12.8
- **signature dictionary**: PDF data structure (type dictionary) as described in ISO 32000-1 [1], clause 12.8.1, table 252
- **signer**: entity that creates an electronic signature
- **validation data**: data used by a verifier to determine signature validity (e.g., certificates, CRLs, OCSP responses)
- **verifier**: entity that validates an electronic signature
- **may**: course of action is permissible within a profile
- **shall**: definition is an absolute requirement of a profile; must be strictly followed for conformance
- **should**: among several possibilities one is recommended as particularly suitable; implementation may choose a different course if fully understood

### Abbreviations
- CAdES: CMS Advanced Electronic Signature (see TS 101 733 [2])
- CMS: Cryptographic Message Syntax (RFC 3852 [5])
- CRL: Certificate Revocation List
- LTV: Long Term Validation
- MDP: Modification Detection Permissions
- OCSP: Online Certificate Status Protocol
- PAdES: PDF Advanced Electronic Signature

## 4 General Features
### 4.1 PDF Signatures
- Digital signatures in ISO 32000-1 [1] support: adding a signature immediately, providing placeholder fields, and checking validity.
- Signature value is encoded as CMS [5] or related format (PKCS #7 [4] or CAdES [2]).
- Digest computed over ByteRange of file excluding the PDF Signature itself (normative requirement in Parts 2 and 3).
- Contents entry holds the PDF Signature binary value; placeholder written as zeros then filled.

### 4.2 PDF Signature Types
- **Certification signatures** with MDP (ISO 32000-1 [1] clause 12.8.4) allow limited modifications (form fill-in, commenting) without invalidating original signature.
- **Usage Rights** (clause 12.8.2.3) enhance document rights using signature to ensure integrity.

### 4.3 PDF Signature Handlers
- Filter/SubFilter pair defines signature handler and encoding.
- Two encodings used: PKCS #7 [4] (Part 2) and CAdES [2] (Part 3).

### 4.4 PDF Serial Signatures
- Only one signer per PDF signature (single SignerInfo). Multiple signatures are serial; each signs previous bytes plus modifications.
- Parallel signatures not supported.
- **shall**: only a single signer per PDF signature (from "there shall only be a single signer").

### 4.5 Validation and Time-stamping
- Use of time-stamping and revocation information varies by profile; see other parts.

### 4.6 ISO 19005-1 (PDF/A-1)
- PDF/A-1 is a subset for long-term archiving; lacks embedded revocation and time-stamping features (but not forbidden). PDF/A-2 (based on ISO 32000-1) expected in 2010, supporting full digital signature features.

### 4.7 Signatures on XML Content in PDF
- XML data (e.g., XFA forms) may be signed with XAdES [3]. Once signed with a PDF signature, XAdES cannot be extended for long-term validation unless XFA is used (see clauses 5.5-5.8).

## 5 Profiles
### 5.1 Part 2: PAdES Basic (ISO 32000-1 based)
- **Description**: Profile per TS 102 778-2 [6]; uses PKCS #7 signature.
- **Features**:
  - Supports serial signatures
  - Recommends inclusion of a signature time-stamp
  - Recommends inclusion of revocation information
  - Protects integrity and authenticates signatory
  - Optionally includes reason, location, contact info
  - Legal content attestation for PDF capabilities

### 5.2 Part 3: PAdES Enhanced - PAdES-BES
- **Description**: Uses CAdES-BES (TS 101 733 [2]) with optional time-stamp (CAdES-T).
- **Features**:
  - Signature encoded as CAdES-BES
  - **must include** signing certificate reference
  - Recommends signature time-stamp (CAdES-T)
  - Supports serial signatures
  - Optionally includes CAdES simple attributes (Signing Time, Signer Attributes, Content-Timestamp)
  - Legal content attestation

### 5.3 Part 3: PAdES Enhanced - PAdES-EPES
- **Description**: Based on CAdES-EPES [2]; adds signature policy identifier and optional commitment type.
- **Features** (same as BES plus):
  - **must include** signing certificate reference
  - **must include** signature policy identifier
  - Optionally includes signature time-stamp (CAdES-T)
  - Optionally includes commitment type indication
  - Supports serial signatures

### 5.4 Part 4: Long Term - PAdES-LTV
- **Description**: Supports long-term validation; applicable with PAdES-CMS, BES, or EPES.
- **Features**:
  - Adds validation data (certificates, OCSP, CRLs) to PDF
  - Adds document time-stamp protecting existing signatures and validation data
  - Further validation data and time-stamps may be added over time

### 5.5 Part 5: Basic XAdES for XML Documents in PDF Containers
- **Description**: XAdES-signed XML (e.g., UBL) embedded in PDF; basic forms (XAdES-BES, EPES, T).
- **Features**:
  - XML signed data independent of PDF container
  - XAdES signatures provide integrity/authenticity; time-stamp enables non-repudiation
  - Supports serial (countersignatures) and parallel signatures

### 5.6 Part 5: Long-term XAdES for XML Documents in PDF Containers
- **Description**: Upgrades basic XAdES forms to evolved forms (XAdES-C, X, XL, A).
- **Features**:
  - Long-term signature production
  - Signature encoded as XAdES-C, X, XL, A (TS 101 903 [3])

### 5.7 Part 5: Basic XAdES on XFA Forms
- **Description**: XAdES signatures for dynamic XFA forms; covers signing XFA data or XML content.
- **Features**:
  - Signs XFA data or allowed XML content per XFA specification [i.2]
  - Encoded as XAdES-T, BES, or EPES
  - Supports serial and parallel signatures

### 5.8 Part 5: Long-term Validation XAdES on XFA Forms (XAdES-LTV)
- **Description**: Upgrades XAdES signatures from clause 5.7 to support long-term validation using LTV mechanisms (Annex A of TS 102 778-4 [8]).
- **Features**:
  - Equivalent to XAdES-XL and XAdES-A forms
  - Uses DSS and VRI dictionaries for validation data
