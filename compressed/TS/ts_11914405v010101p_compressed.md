# ETSI TS 119 144-5: PAdES digital signatures - Testing Conformance and Interoperability; Part 5: Testing Conformance of additional PAdES signatures
**Source**: ETSI | **Version**: V1.1.1 | **Date**: 2016-06 | **Type**: Technical Specification  
**Original**: https://www.etsi.org/deliver/etsi_ts/119100_119199/11914405/01.01.01_60/ts_11914405v010101p.pdf

## Scope (Summary)
Defines the set of checks for testing conformance of PAdES signatures against additional PAdES signature profiles as specified in ETSI EN 319 142-2 [3], complementing the baseline checks in ETSI TS 119 144-4 [6]. Does not assess technical validity of cryptographic material.

## 4 Overview
Requirements are classified as:
1. **PAdES_BB**: building block requirements from EN 319 142-1 (tested in TS 119 144-4 [6]).
2. **PAdES_AS**: additional signature requirements from EN 319 142-2 [3].

Tests cover structure, values, interrelationships, and computations. No tests on cryptographic validity. Each test assertion includes: unique ID (prefix `PAdES_AS`), normative source, target, optional prerequisite, predicate, prescription level (mandatory/recommended/permitted), and tag.

## 5 Testing conformance to CMS signatures in PDF profile
### 5.2 Testing signature dictionary elements
- **PAdES_BB/SDC/1** check (as per [6]) **shall** apply.
- **PAdES_AS/SDC/1** (mandatory): For new signatures, applications encode the signature, included in the signature dictionary entry with the key Contents, as a DER-encoded PKCS#7 binary data object. (Source: [3] Clause 4.2.1)
- **PAdES_BB/SDBR/1-2** checks (as per [6]) **shall** apply.
- **PAdES_BB/SDF/1** check (as per [6]) **shall** apply.
- **PAdES_AS/SDSF/1** (mandatory): For new signatures, applications include the string "adbe.pkcs7.detached" in the SubFilter entry of the signature dictionary. (Source: [3] Clause 4.2.2)
- **PAdES_AS/SDSF/2** (recommended): While validating signatures, applications accept the string "adbe.pkcs7.detached" or "adbe.pkcs7.sha1" in the SubFilter entry of the signature dictionary. (Source: [3] Clause 4.2.2)
- **PAdES_AS/SDM/1** (permitted): For new signatures, applications include the claimed time of signing in the M entry of the signature dictionary. (Source: [3] Clause 4)
- **PAdES_BB/SDL/1**, **PAdES_BB/SDR/1-2**, **PAdES_BB/SDCERT/1**, **PAdES_BB/SDCI/1**, **PAdES_BB/SDNAME/1** checks (as per [6]) **shall** apply.

### 5.3 Testing PKCS #7 signature elements
- **PAdES_BB/CER/1-2** checks (as per [6]) **shall** apply.
- **PAdES_AS/CRL/1** (recommended): For new signatures, applications include all crls or ocsp responses needed for certificates validation in the signed attributes of the signature. (Source: [3] Clause 4.2.1)
- **PAdES_AS/TST/1** (recommended): For new signatures, applications include a timestamp in the unsigned attributes of the signature. (Source: [1] Clause 4.2.1)
- **PAdES_AS/STI/1** (permitted): For new signatures, applications include a claimed signing time in the signed attributes of the signature. (Source: [1] Clause 4)
- **PAdES_AS/SL/1** (permitted): For new signatures, applications include a signer-location in the signed attributes of the signature. (Source: [1] Clause 4)

## 6 Testing conformance to PAdES-E-BES and PAdES-E-EPES additional signatures
### 6.2.1 Testing PAdES-E-BES signature dictionary elements
- **PAdES_BB/SDC/1**, **PAdES_BB/SDBR/1-2**, **PAdES_BB/SDF/1**, **PAdES_BB/SDSF/1** checks (as per [6]) **shall** apply.
- **PAdES_AS/SDM/1** (clause 5.1) **shall** apply.
- **PAdES_BB/SDL/1**, **PAdES_BB/SDR/1-2**, **PAdES_BB/SDCERT/1**, **PAdES_BB/SDCI/1**, **PAdES_BB/SDNAME/1** checks (as per [6]) **shall** apply.

### 6.2.2 Testing PAdES-E-BES CMS signature elements
- **PAdES_BB/CER/1-2** checks (as per [6]) **shall** apply.
- **PAdES_AS/TST/1** (clause 5.2) **shall** apply.
- **PAdES_AS/STI/1** (clause 5.2) **shall** apply.
- **PAdES_AS/SL/1** (clause 5.2) **shall** apply.
- **PAdES_BB/ESS/1-2-3-4**, **PAdES_BB/MD/1**, **PAdES_BB/CTY/1-2**, **PAdES_BB/CS/1**, **PAdES_BB/CR/1**, **PAdES_BB/CI/1**, **PAdES_BB/CH/1**, **PAdES_BB/CTI/1**, **PAdES_BB/SL/1**, **PAdES_BB/SA/1**, **PAdES_BB/CTS/1** checks (as per [6]) **shall** apply.

### 6.3.1 Testing PAdES-E-EPES signature dictionary elements
- **PAdES_BB/SDBR/1-2** checks (as per [6]) **shall** apply.

### 6.3.2 Testing PAdES-E-EPES CMS signature elements
- **PAdES_AS/SPID/1** (mandatory): For new signatures, applications include signature-policy-identifier attribute in CMS signature. (Source: [3] Clause 5.4)

## 7 Testing conformance to PAdES-E-LTV additional signatures
### 7.2 Testing DSS dictionary
- **PAdES_BB/DSS/1-2-3-4-5** checks (as per [6]) **shall** apply.

### 7.3 Testing DTS dictionary
- **PAdES_BB/DTS/2-3-4-5** checks (as per [6]) **shall** apply.

## 8 Testing conformance to profiles for XAdES signatures signing XML content in PDF specification
- PAdES signatures conformant to these profiles **shall** fulfil the requirements specified in clause 5 of ETSI TS 119 134-4 [4].
