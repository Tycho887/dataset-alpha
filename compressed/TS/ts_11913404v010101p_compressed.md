# ETSI TS 119 134-4 V1.1.1: XAdES digital signatures - Testing Conformance and Interoperability; Part 4: Testing Conformance of XAdES baseline signatures
**Source**: ETSI | **Version**: V1.1.1 | **Date**: 2016-06 | **Type**: Normative
**Original**: [ETSI TS 119 134-4](http://www.etsi.org/standards-search)

## Scope (Summary)
Defines the set of checks to test conformance of XAdES signatures against XAdES baseline signatures as specified in ETSI EN 319 132-1 [1]. Does not assess technical validity of signatures. Specifies structural and computational checks, not cryptographic validity.

## Definitions and Abbreviations
- **XAdES**: XML Advanced Electronic Signatures
- **XAdES-B-B**: Baseline signature with basic building blocks
- **XAdES-B-T**: Baseline signature with time-stamp
- **XAdES-B-LT**: Baseline signature with long-term validation data
- **XAdES-B-LTA**: Baseline signature with archive time-stamp
- **TA**: Test Assertion
- **XMLDSIG / ds**: XML Signature namespace `http://www.w3.org/2000/09/xmldsig#`
- **xades**: namespace `http://uri.etsi.org/01903/v1.3.2#`
- **xadesv141**: namespace `http://uri.etsi.org/01903/v1.4.1#`

## Overview
- Tests are defined as test assertions following OASIS Test Assertions Model [i.1].
- Three groups of requirements: XAdES_BS (baseline-specific), XAdES_BB (common building blocks), XAdES_SCH (schema-derived).
- Test assertions for baseline signatures are the addition of XAdES_BB and XAdES_BS.
- Each assertion has: unique ID, normative source, target, optional prerequisite, predicate, prescription level (mandatory/preferred/permitted), and tag.

## 5 Testing conformance to B-B level of XAdES signatures

### 5.1 General
- Clause 5 specifies assertions not directly derived from XAdES XML schema for XAdES-B-B signatures.
- Clause 5.2: assertions on XML Signature elements profiled by [1]. Clause 5.3: assertions on qualifying properties relevant to B-B.
- XAdES-B-B signatures **should not** incorporate certain properties (e.g., SignatureTimeStamp, CompleteCertificateRefsV2, etc.), but if present they **shall** be tested as indicated in cross-references.

### 5.2 Testing XML Signature elements and containers

#### 5.2.1 Testing XML Signature elements

##### 5.2.1.1 Testing ds:Signature element
- **XADES_BS/XMLSIG/DSSIG/1** (mandatory): The signature has one or more ds:Object children.

##### 5.2.1.2 Testing ds:Reference element

**Common assertions (XAdES_BB)**:
- **XADES_BB/XMLSIG/REF/1** (mandatory): The Type attribute of ds:Reference referencing xades:SignedProperties is "http://uri.etsi.org/01903#SignedProperties".
- **XADES_BB/XMLSIG/REF/2** (mandatory): The digest value in ds:Reference equals the computed value per XMLDSIG [3] clause 4.4.3.2.

**Specific to baseline (XAdES_BS)**:
- **XADES_BS/XMLSIG/REF/TR/1** (preferred): Range of transformation algorithms in new signatures is as identified in [1] clause 6.3.

##### 5.2.1.3 Testing ds:CanonicalizationMethod element
- **XADES_BS/XMLSIG/REF/CAN/1** (preferred): Algorithm attribute for new signatures is one of `http://www.w3.org/2006/12/xml-c14n11`, `http://www.w3.org/2001/10/xml-exc-c14n#`, `http://www.w3.org/TR/2001/REC-xml-c14n-20010315`.
- **XADES_BS/XMLSIG/REF/CAN/2** (permitted): For legacy signatures, additional allowed values include `#WithComments` variants.

##### 5.2.1.4 Testing ds:SignatureValue element
- **XADES_BS/XMLSIG/SIGVAL/TR/1** (mandatory): The value in ds:SignatureValue is cryptographically correct.

##### 5.2.1.5 Testing ds:KeyInfo element

**Common**:
- **XADES_BB/XMLSIG/KEYINFO/1** (mandatory): Any ds:X509Certificate descendant contains base-64 encoding of X.509 certificate.

**Specific to baseline**:
- **XADES_BS/XMLSIG/KEYINFO/1** (mandatory): ds:KeyInfo is present.
- **XADES_BS/XMLSIG/KEYINFO/2** (mandatory): ds:KeyInfo includes the signing certificate in one of its descendant X509Certificate elements.

#### 5.2.2 Testing containers of XAdES qualifying properties

##### 5.2.2.1 Testing incorporation of qualifying properties
- **XADES_BS/PROPERTIES/INCORPORATION/1** (mandatory): One and exactly one xades:QualifyingProperties descendant element.
- **XADES_BS/PROPERTIES/INCORPORATION/2** (mandatory): xades:QualifyingProperties is child of a ds:Object that is child of ds:Signature.
- **XADES_BS/PROPERTIES/INCORPORATION/3** (mandatory): No ds:Object includes xades:QualifyingPropertiesReference.

##### 5.2.2.2 Testing xades:QualifyingProperties

**Common**:
- **XADES_BB/QP/1** (mandatory): Not an empty element.
- **XADES_BB/QP/2** (mandatory): Target attribute references the Id of the ds:Signature root element.

**Specific**:
- **XADES_BS/QP/1** (mandatory): Contains exactly one xades:SignedProperties child.

##### 5.2.2.3 Testing xades:SignedProperties

- **XADES_BS/SIGPROP/1** (mandatory): Contains one xades:SignedSignatureProperties child.
- **XADES_BS/SIGPROP/3** (mandatory, prerequisite: countersignature signs additional data objects): Contains one xades:SignedDataObjectProperties child.
- **XADES_BS/SIGPROP/2** (mandatory, prerequisite: signature does not countersign): Contains one xades:SignedDataObjectProperties child.

##### 5.2.2.4 Testing xades:SignedSignatureProperties

- **XADES_BS/SIGSIGPROP/1** (mandatory): Contains xades:SigningCertificateV2.
- **XADES_BS/SIGSIGPROP/2** (mandatory): Contains xades:SigningTime.

##### 5.2.2.5 Testing xades:SignedDataObjectProperties

**Common**:
- **XADES_BB/SIGDOPROP/1** (mandatory): Not an empty element.

**Specific**:
- **XADES_BS/SIGDOPROP/1** (mandatory, signature does not countersign): One xades:DataObjectFormat per signed data object except xades:SignedProperties.
- **XADES_BS/SIGDOPROP/2** (mandatory, countersignature case): One xades:DataObjectFormat per additional signed data object.

##### 5.2.2.6 Testing xades:UnSignedProperties

**Common**:
- **XADES_BB/UNSIGPROP/1** (mandatory): Not an empty element.

**Specific**:
- **XADES_BS/UNSIGPROP/1** (mandatory): Contains 0 or 1 xades:UnSignedProperties element.

##### 5.2.2.7 Testing xades:UnSignedSignatureProperties

**Common**:
- **XADES_BB/UNSIGSIGPROP/1** (mandatory): Not an empty element.

**Specific**:
- **XADES_BS/UNSIGSIGPROP/1** (mandatory): Contains 0 or 1 xades:UnSignedSignatureProperties element.

##### 5.2.2.8 Testing xades:UnSignedDataObjectProperties

**Common**:
- **XADES_BB/UNSIGDOPROP/1** (mandatory): Not an empty element.
- **XADES_BB/UNSIGDOPROP/2** (mandatory): A XAdES signature does not incorporate any xades:UnSignedDataObjectProperties element.

### 5.3 Testing XAdES qualifying properties

#### 5.3.1 Testing xades:SigningTime
- No additional assertions beyond Annex A.

#### 5.3.2 Testing xades:SigningCertificateV2

**Common**:
- **XADES_BB/SIGCERTV2/1** (mandatory): First xades:Cert child contains base-64 digest of DER-encoded signing certificate.
- **XADES_BB/SIGCERTV2/2** (permitted): May contain other xades:Cert elements.
- **XADES_BB/SIGCERTV2/3** (permitted): Other xades:Cert children contain digests of other certificates in the signing path.

**Specific**:
- **XADES_BS/SIGCERTV2/1** (mandatory): Does not contain Cert children's URI attribute.
- **XADES_BS/SIGCERTV2/2** (preferred): Does not include IssuerSerialV2 Cert child.

#### 5.3.3 Testing xades:CommitmentTypeIndication

**Common**:
- **XADES_BB/CTI/1** (mandatory): Any xades:ObjectReference references a ds:Reference in ds:SignedInfo or signed ds:Manifest.
- **XADES_BB/CTI/2** (mandatory): No xades:ObjectReference references the ds:Reference referencing xades:SignedProperties.
- **XADES_BB/CTI/3** (mandatory): Identifier child does not contain Qualifier attribute.

#### 5.3.4 Testing xades:DataObjectFormat

**Common**:
- **XADES_BB/DOF/1** (mandatory): ObjectReference attribute references a ds:Reference child of ds:SignedInfo or signed ds:Manifest.
- **XADES_BB/DOF/2** (mandatory): ObjectReference does not reference the ds:Reference pointing to xades:SignedProperties.
- **XADES_BB/DOF/3** (mandatory): ObjectReference does not reference a ds:Reference within a countersignature.
- **XADES_BB/DOF/4** (mandatory): If xades:MimeType is present and referenced ds:Object has MimeType attribute, values must match.
- **XADES_BB/DOF/5** (mandatory): Similarly for Encoding.
- **XADES_BB/DOF/6** (mandatory): If Qualifier="OIDAsURN", Identifier child must be an OID encoded as URN per RFC 3061.

**Specific**:
- **XADES_BS/DOF/1** (mandatory, not a countersignature): One xades:DataObjectFormat for each signed data object except xades:SignedProperties.
- **XADES_BS/DOF/2** (mandatory, countersignature): One per signed data object except SignedProperties and countersigned signatures.
- **XADES_BS/DOF/3** (mandatory): Contains one xades:MimeType child.

#### 5.3.5 Testing xades:SignatureProductionPlaceV2

- **XADES_BB/SIGPRODPLV2/1** (mandatory): Not an empty element.

#### 5.3.6 Testing xades:SignerRoleV2

- **XADES_BB/SIGROLEV2/1** (mandatory): Not empty.
- **XADES_BB/SIGROLE/2** (mandatory): Any xades:X509AttributeCertificate encapsulates base-64 encoding of DER-encoded X.509 attribute certificate.
- **XADES_BB/SIGROLE/3** (mandatory): If xades:SignedAssertion is XML node set, it must be well-formed.
- **XADES_BB/SIGROLE/4** (mandatory): Any xades:SignedAssertion element is signed.

#### 5.3.7 Testing xades:CounterSignature

- **XADES_BB/COUNTSIG/2** (mandatory): Contains ds:Reference referencing the ds:SignatureValue of the countersigned signature.
- **XADES_BB/COUNTSIG/3** (mandatory): ds:SignatureValue child is cryptographically correct.
- The countersignature itself shall be tested against all assertions for baseline signatures.

#### 5.3.8 Testing xades:AllDataObjectsTimeStamp

- **XADES_BB/ADOTST/1** (mandatory): Does not contain any xades:Include child.
- **XADES_BB/ADOTST/2** (mandatory): Encapsulates one or more time-stamp tokens.
- **XADES_BB/ADOTST/3** (mandatory): Uses implicit mechanism per [1] clause 5.1.4.4.1.
- **XADES_BB/ADOTST/4** (mandatory): Message imprint input is concatenation of all ds:Reference objects after processing (except the one referencing SignedProperties), in order.

#### 5.3.9 Testing xades:IndividualDataObjectsTimeStamp

- **XADES_BB/IDOTST/1** (mandatory): Contains one or more xades:Include children.
- **XADES_BB/IDOTST/2** (mandatory): Encapsulates one or more time-stamp tokens.
- **XADES_BB/IDOTST/3** (mandatory): Any xades:Include references a ds:Reference in ds:SignedInfo or signed ds:Manifest.
- **XADES_BB/IDOTST/4** (mandatory): Referenced ds:Reference does not point to xades:SignedProperties.
- **XADES_BB/IDOTST/5** (mandatory): referencedData attribute of any xades:Include is "true".
- **XADES_BB/IDOTST/6** (mandatory): Message imprint input is concatenation of objects from referenced ds:References.

#### 5.3.10 Testing xades:SignaturePolicyIdentifier

- **XADES_BB/SIGPOLID/1** (mandatory): xades:SigPolicyHash contains hash of SigPolicyId after processing transforms if present.
- **XADES_BB/SIGPOLID/2** (mandatory): If a transform URI is `http://uri.etsi.org/01903/v1.3.2/SignaturePolicy/SPDocDigestAsInSpecification`, then xadesv141:SPDocSpecification qualifier must be present.
- **XADES_BB/SIGPOLID/3** (permitted): May contain one or more qualifiers of the same type.

##### 5.3.10.2 SPURI
- **XADES_BB/SIGPOLID/SPURI/1** (mandatory): The URI reference contains the signature policy document.

##### 5.3.10.3 SPDocSpecification
- **XADES_BB/SIGPOLID/SPDOCSPEC/1** (mandatory, OID): Identifier contains URN encoding OID per RFC 3061 and QualifierType="OIDAsURN".
- **XADES_BB/SIGPOLID/SPDOCSPEC/2** (mandatory, URI): Identifier is the URI, QualifierType absent.

#### 5.3.11 Testing xadesv141:SignaturePolicyStore

- **XADES_BB/SIGPOLST/1** (mandatory): If no xades:SignaturePolicyIdentifier, then no xadesv141:SignaturePolicyStore.
- **XADES_BB/SIGPOLST/2** (mandatory): If SignaturePolicyIdentifier present but no SigPolicyHash descendant, then no SignaturePolicyStore.
- **XADES_BB/SIGPOLST/3** (mandatory): If empty SigPolicyHash, then no SignaturePolicyStore.
- **XADES_BB/SIGPOLST/4** (permitted): If non-empty SigPolicyHash, may incorporate SignaturePolicyStore.
- **XADES_BB/SIGPOLST/5** (mandatory): The place referenced by xades:SigPolDocLocalURI contains the signature policy document.

#### 5.3.12 Testing xadesv141:CompleteCertificateRefsV2

**Common**:
- **XADES_BB/CCERTREFSV2/1** (mandatory): Does not contain reference of the signing certificate.
- **XADES_BB/CCERTREFSV2/2** (mandatory): If CertificateValues, AttrAuthoritiesCertValues, or ArchiveTimeStamp are present, then all referenced certificates must be present elsewhere.

**Specific**:
- **XADES_BS/CCERTREFSV2/1** (preferred): Not present in B-B signatures.
- **XADES_BS/CCERTREFSV2/2** (mandatory): At most one instance.
- **XADES_BS/CCERTREFSV2/3** (preferred): Does not include IssuerSerialV2 child of Cert.

#### 5.3.13 Testing xadesv141:AttributeCertificateRefsV2

**Common**:
- **XADES_BB/ATTCERTREFSV2/1** (preferred): Does not contain references present in CompleteCertificateRefsV2 or SigningCertificateV2.
- **XADES_BB/ATTCERTREFSV2/2** (mandatory): If CertificateValues, AttrAuthoritiesCertValues, or ArchiveTimeStamp present, all referenced certificates must be present.
- **XADES_BB/ATTCERTREFSV2/3** (permitted): If attribute certificate or signed assertion incorporated, property may be present.

**Specific**:
- **XADES_BS/ATTCERTREFSV2/1** (preferred): Not present in B-B signatures.
- **XADES_BS/ATTCERTREFSV2/2** (mandatory): At most one instance.
- **XADES_BS/ATTCERTREFSV2/3** (preferred): Does not include IssuerSerialV2 Cert child.

#### 5.3.14 Testing xades:CompleteRevocationRefs

**Common**:
- **XADES_BB/CREVREFS/1** (mandatory): Contains reference to revocation value for signing certificate.
- **XADES_BB/CREVREFS/2** (mandatory): Contains references for CA certificates in the signing path.
- **XADES_BB/CREVREFS/3** (mandatory): Does not contain references for trust anchor.
- **XADES_BB/CREVREFS/4** (mandatory): xades:byKey child of ResponderID contains base-64 DER-encoded ‘byKey’ per RFC 6960.
- **XADES_BB/CREVREFS/5** (mandatory): xades:ProducedAt matches the time in the referenced OCSP response.
- **XADES_BB/CREVREFS/6** (preferred): OCSPRef contains xades:DigestAlgAndValue.
- **XADES_BB/CREVREFS/7** (mandatory): Not empty.
- **XADES_BB/CREVREFS/8** (mandatory): If CertificateValues, AttrAuthoritiesCertValues, or ArchiveTimeStamp present, all referenced revocation data must be present elsewhere.

**Specific**:
- **XADES_BS/CREVREFS/1** (preferred): Not present in B-B signatures.
- **XADES_BS/CREVREFS/2** (mandatory): At most one instance.

#### 5.3.15 Testing xades:AttributeRevocationRefs

**Common**:
- **XADES_BB/ATTREVREFS/1** (mandatory): Missing revocation references for attribute certificate signers must be present here if not in CompleteRevocationRefs.
- **XADES_BB/ATTREVREFS/2** (mandatory): Does not contain trust anchor references.
- **XADES_BB/ATTREVREFS/3** (mandatory): Same as CREVREFS/4.
- **XADES_BB/ATTREVREFS/4** (mandatory): Same as CREVREFS/5.
- **XADES_BB/ATTREVREFS/5** (preferred): OCSPRef contains DigestAlgAndValue.
- **XADES_BB/ATTREVREFS/6** (mandatory): Not empty.
- **XADES_BB/ATTREVREFS/7** (mandatory): If CertificateValues, AttrAuthoritiesCertValues, or ArchiveTimeStamp present, all referenced revocation data must be present elsewhere.

**Specific**:
- **XADES_BS/ATTREVREFS/1** (preferred): Not present in B-B.
- **XADES_BS/ATTREVREFS/2** (mandatory): At most one instance.

#### 5.3.16 Testing xadesv141:SigAndRefsTimeStampV2

**Common**:
- **XADES_BB/SIGREFSTSTV2/1** (mandatory): If no xades:Include, message imprint input is canonicalized concatenation of ds:SignatureValue, SignatureTimeStamp, CompleteCertificateRefsV2, CompleteRevocationRefs, AttributeCertificateRefsV2 (if present), AttributeRevocationRefs (if present), in order.
- **XADES_BB/SIGREFSTSTV2/2** (mandatory): Time in this token shall be later than time in IndividualDataObjectsTimeStamp, AllDataObjectsTimeStamp, and SignatureTimeStamp.

**Specific**:
- **XADES_BS/SIGREFSTSTV2/1** (preferred): Not present in B-B.
- **XADES_BS/SIGREFSTSTV2/2** (mandatory): Uses implicit mechanism (no xades:Include).

#### 5.3.17 Testing xadesv141:RefsOnlyTimeStampV2

**Common**:
- **XADES_BB/REFSONTSTV2/1** (mandatory): If no xades:Include, message imprint input is canonicalized concatenation of CompleteCertificateRefsV2, CompleteRevocationRefs, AttributeCertificateRefsV2 (if present), AttributeRevocationRefs (if present), in order.
- **XADES_BB/REFSONTSTV2/2** (mandatory): Time later than in IndividualDataObjectsTimeStamp, AllDataObjectsTimeStamp, SignatureTimeStamp.

**Specific**:
- **XADES_BS/REFSONTSTV2/1** (preferred): Not present in B-B.
- **XADES_BS/REFSONTSTV2/2** (mandatory): Uses implicit mechanism.

## 6 Testing conformance to B-T level of XAdES signatures

### 6.1 General requirements
- Apply assertions from clause 6 plus clauses 5.2 and 5.3 unless stated otherwise.
- XAdES-B-T signatures **should not** incorporate certain properties (CompleteCertificateRefsV2, CompleteRevocationRefs, etc.) but if present, test per clause 5.1.

### 6.2 Testing xades:SignatureTimeStamp

**Common**:
- **XADES_BB/SIGTST/1** (mandatory): Message imprint input is canonicalized ds:SignatureValue.
- **XADES_BB/SIGTST/2** (mandatory): If IndividualDataObjectsTimeStamp and/or AllDataObjectsTimeStamp present, time here is later than time in those tokens.

**Specific**:
- **XADES_BS/SIGTST/1** (mandatory): SignatureTimeStamp is present.
- **XADES_BS/SIGTST/2** (mandatory): Encapsulates one time-stamp token.

## 7 Testing conformance to B-LT level of XAdES signatures

### 7.1 General requirements
- Apply clauses 7, 5.2, 5.3, 6.2 unless stated otherwise.

#### 7.1.2 Properties containing references to validation data
- For B-LT and B-LTA, the following properties **shall not** be present: CompleteCertificateRefsV2, AttributeCertificateRefsV2, CompleteRevocationRefs, AttributeRevocationRefs, SigAndRefsTimeStampV2, RefsOnlyTimeStampV2.
- The corresponding test assertions from clause 5.3.12–5.3.17 **shall not** apply.
- Instead, the following mandatory assertions apply:
  - **XADES_BS/BLT_BLTA/CCERTREFSV2/1**: CompleteCertificateRefsV2 not present.
  - **XADES_BS/BLT_BLTA/ATTCERTREFSV2/1**: AttributeCertificateRefsV2 not present.
  - **XADES_BS/BT_BLT_BLTA/CREVREFS/1**: CompleteRevocationRefs not present.
  - **XADES_BS/BLT_BLTA/ATTREVREFS/1**: AttributeRevocationRefs not present.
  - **XADES_BS/BLT_BLTA/SIGREFSTSTV2/1**: SigAndRefsTimeStampV2 not present.
  - **XADES_BS/BLT_BLTA/REFSONTSTV2/1**: RefsOnlyTimeStampV2 not present.

#### 7.1.3 Properties from upper levels
- XAdES-B-LT **should not** incorporate ArchiveTimeStamp or RenewedDigests. If present, test per clause 5.1.

### 7.2 Testing xades:CertificateValues

**Common**:
- **XADES_BB/CERTVALS/1** (mandatory): Any xades:EncapsulatedX509Certificate child contains base-64 DER-encoded X.509 certificate.

**Specific**:
- **XADES_BS/CERTVALS/1** (mandatory): If other elements miss required certificates for validation, CertificateValues must be present.
- **XADES_BS/CERTVALS/2** (mandatory): At most one instance.
- **XADES_BS/CERTVALS/3** (preferred): Does not include certificates present elsewhere.

### 7.3 Testing xades:RevocationValues

**Common**:
- **XADES_BB/REVVALS/1** (mandatory): Each EncapsulatedCRLValue is base-64 DER-encoded CRL.
- **XADES_BB/REVVALS/2** (mandatory): Each EncapsulatedOCSPValue is base-64 DER-encoded OCSPResponse per RFC 6960.

**Specific**:
- **XADES_BS/REVVALS/1** (mandatory): If other elements miss required revocation values, RevocationValues must be present.
- **XADES_BS/REVVALS/2** (mandatory): At most one instance.
- **XADES_BS/REVVALS/3** (preferred): Does not include revocation values present elsewhere.

### 7.4 Testing xades:AttrAuthoritiesCertValues

**Common**:
- **XADES_BB/ATTCERTVALS/1** (mandatory): EncapsulatedX509Certificate child is base-64 DER-encoded certificate.

**Specific**:
- **XADES_BS/ATTCERTVALS/1** (mandatory): If signature incorporates attribute certificates/signed assertions and other elements miss certificates needed for validation, AttrAuthoritiesCertValues must be present.
- **XADES_BS/ATTCERTVALS/2** (mandatory): At most one instance.
- **XADES_BS/ATTCERTVALS/3** (preferred): Does not include certificates present elsewhere.

### 7.5 Testing xades:AttributeRevocationValues

**Common**:
- **XADES_BB/ATTREVVALS/1** (mandatory): EncapsulatedCRLValue is base-64 DER-encoded CRL.
- **XADES_BB/ATTREVVALS/2** (mandatory): EncapsulatedOCSPValue is base-64 DER-encoded OCSPResponse.

**Specific**:
- **XADES_BS/ATTREVVALS/1** (mandatory): If signature incorporates attribute certificates/signed assertions and other elements miss required revocation values, AttributeRevocationValues must be present.
- **XADES_BS/ATTREVVALS/2** (mandatory): At most one instance.
- **XADES_BS/ATTREVVALS/3** (preferred): Does not include revocation data present elsewhere.

### 7.6 Testing xadesv141:TimeStampValidationData

**Common**:
- **XADES_BB/TSTVALDAT/1** (mandatory): EncapsulatedX509Certificate is base-64 DER-encoded certificate.
- **XADES_BB/TSTVALDAT/2** (mandatory): EncapsulatedCRLValue is base-64 DER-encoded CRL.
- **XADES_BB/TSTVALDAT/3** (mandatory): EncapsulatedOCSPValue is base-64 DER-encoded OCSPResponse.
- **XADES_BB/TSTVALDAT/4** (mandatory): If direct incorporation, property is either first child of UnsignedSignatureProperties or immediately after a time-stamp token container.
- **XADES_BB/TSTVALDAT/5** (preferred): If immediately after a time-stamp token container, URI attribute not present.
- **XADES_BB/TSTVALDAT/6** (preferred): Otherwise, URI attribute present and references the specific container whose validation material it holds.

## 8 Testing conformance to B-LTA level of XAdES signatures

### 8.1 General requirements
- Apply clauses 8, 5.2, 5.3, 6.2, 7.1.2, 7.2–7.6 unless stated otherwise.

### 8.2 Testing xadesv141:ArchiveTimeStamp

#### 8.2.1 Common tests
- **XADES_BB/ATSTV141/1** (mandatory): If no xades:Include, message imprint input is as specified in [1] clause 5.5.2.2.

#### 8.2.2 Specific to baseline
- **XADES_BS/ATST/1** (mandatory): ArchiveTimeStamp is present in B-LTA signatures.
- **XADES_BS/ATST/2** (mandatory): Uses implicit mechanism (no xades:Include).
- **XADES_BS/ATST/3** (mandatory): Time in any ArchiveTimeStamp token is later than time in IndividualDataObjectsTimeStamp (if present), AllDataObjectsTimeStamp (if present), SignatureTimeStamp, and any preceding ArchiveTimeStamp.

### 8.3 Testing xadesv141:RenewedDigests

**Common**:
- **XADES_BB/RENDIG/1** (mandatory): Any xadesv141:RecomputedDigestValue contains base-64 digest computed with ds:DigestMethod on the ds:Reference identified by the Order attribute, per XMLDSIG reference processing model.
- **XADES_BB/RENDIG/2** (mandatory): If signature does not contain any signed ds:Manifest referencing detached data objects, RenewedDigests is not present.

**Specific**:
- **XADES_BS/RENDIG/1** (permitted): RenewedDigests property may be present.

## Annex A (normative): Test assertions derived from XML Schema

### A.0 Introduction
- This annex specifies structural tests based on XAdES XML Schema. Notation: children lists, attributes, modifiers (?, +, *, noOrder, choice).

### A.1 Testing auxiliary types contents

#### A.1.2 Testing xades:ObjectIdentifierType instances
- Structure: xades:Identifier, xades:Description?, xades:DocumentationReferences?.
- No attributes. Identifier has no children, valid URI, optional Qualifier attribute (values "OIDAsURI" or "OIDAsURN").

#### A.1.3 Testing xades:EncapsulatedPKIDataType
- No children, optional Id and Encoding attributes (Encoding values: DER, BER, CER, PER, XER URIs). Content must be valid Base64.

#### A.1.4 Testing xades:XAdESTimeStampType
- Children: xades:Include*, ds:CanonicalizationMethod?, choice(xades:EncapsulatedTimeStamp, xades:XMLTimeStamp)+. Optional Id attribute.
