# ETSI TS 119 412-1 V1.4.1: Electronic Signatures and Infrastructures (ESI); Certificate Profiles; Part 1: Overview and common data structures

## Scope (Summary)
Provides an overview of Recommendation ITU-T X.509 | ISO/IEC 9594‑8 based certificate profiles and QCStatements for EU Qualified Certificates specified in other parts of ETSI EN 319 412. Specifies common data structures referenced from those parts. Supports both Regulation (EU) No 910/2014 and use of certificates in a wider international context.

## Definitions and Abbreviations
- **EU Qualified Certificate**: qualified certificate that is stated to be in accordance with annex I, III or IV of Regulation (EU) No 910/2014 or annex I of Directive 1999/93/EC, whichever is in force at the time of issuance.
- **short-term certificate**: certificate whose validity period (notBefore through notAfter, inclusive) is shorter than the maximum time to process a revocation request as specified in the certificate practice statement.
- **ASN.1** : Abstract Syntax Notation 1

## 4 ETSI EN 319 412 certificate profiles
### 4.1 General approach
All certificate profiles specified in ETSI EN 319 412 are based upon IETF RFC 5280 for generic profiling of Recommendation ITU‑T X.509 | ISO/IEC 9594‑8. Profiles specify both EU Qualified Certificates and non-qualified certificates. Reference is made to ETSI EN 319 412‑5 for QCStatements.

### 4.2 Overview of other parts of ETSI EN 319 412
#### 4.2.1 ETSI EN 319 412-2 (natural persons)
Specifies certificate content requirements for TSPs issuing certificates to natural persons. Facilitates interoperability for digital signatures, peer entity authentication, data authentication, data confidentiality. Profile for both EU Qualified and non-qualified certificates. When issued as EU Qualified, references EN 319 412‑5.

#### 4.2.2 ETSI EN 319 412-3 (legal persons)
Analogous to part 2, but for legal persons.

#### 4.2.3 ETSI EN 319 412-4 (web site certificates)
For website certificates accessed via TLS (IETF RFC 5246). Issued to legal or natural persons. Profile for both EU Qualified and non-qualified. References EN 319 412‑5 when EU Qualified.

#### 4.2.4 ETSI EN 319 412-5 (QCStatements)
Requirements on QCStatements as required for qualified certificates in parts 2–4. QCStatements defined in clause 4.3 may be applied outside the EU; other requirements in clause 4 are specific to Regulation (EU) No 910/2014 but may be adapted.

## 5 Common data structures
### 5.1 Semantics identifiers
#### 5.1.1 General
Subject and issuer names can include attributes without semantic disclosure (e.g., `serialNumber`, `organizationIdentifier`). IETF RFC 3739 defines `qcStatement‑2` with `SemanticsInformation`. Semantics identifiers use 2‑character ISO 3166 country codes. Trans‑national codes (EU, UN) may be used. User‑defined country code `'XG'` may be used for global schemes.  
**Identifiers using user‑defined country codes shall be interpreted under the context of the certificate issuer** (no guarantee of uniqueness across issuers). Unassigned codes should not be used.  
NOTE: Semantics identifiers are defined for subject field attributes only; no mechanism for issuer field.

#### 5.1.2 ASN.1 module
The syntax for natural and legal person semantics identifiers **shall** be as defined by the following ASN.1 module:

```asn1
ETSISemanticsIdentifierMod { itu-t(0) identified-organization(4) etsi(0) id-cert-profile(194121) id-mod(0) id-mod-semantics-identifier(0) v2(1) }
DEFINITIONS EXPLICIT TAGS ::= BEGIN
-- Semantics identifiers
id-etsi-qcs-semantics-identifiers OBJECT IDENTIFIER ::= { itu-t(0) identified-organization(4) etsi(0) id-cert-profile(194121) 1 }
-- Semantics identifier for natural person identifier
id-etsi-qcs-semanticsId-Natural OBJECT IDENTIFIER ::= { id-etsi-qcs-semantics-identifiers 1 }
-- Semantics identifier for legal person identifier
id-etsi-qcs-SemanticsId-Legal OBJECT IDENTIFIER ::= { id-etsi-qcs-semantics-identifiers 2 }
-- Semantics identifier for eIDAS natural person identifier
id-etsi-qcs-semanticsId-eIDASNatural OBJECT IDENTIFIER ::= { id-etsi-qcs-semantics-identifiers 3 }
-- Semantics identifier for eIDAS legal person identifier
id-etsi-qcs-SemanticsId-eIDASLegal OBJECT IDENTIFIER ::= { id-etsi-qcs-semantics-identifiers 4 }
END
```

#### 5.1.3 Natural person semantics identifier
When `id‑etsi‑qcs‑SemanticsId‑Natural` is included, any `serialNumber` attribute in the subject field **shall** contain:  
- 3‑character identity type reference;  
- 2‑character ISO 3166 country code;  
- hyphen‑minus `-`;  
- identifier (according to country and identity type reference).  

The three initial characters **shall** be one of:  
1) `"PAS"` – passport number.  
2) `"IDC"` – national identity card number.  
3) `"PNO"` – (national) personal number.  
4) `"TAX"` – personal tax reference number (deprecated; use `"TIN"`).  
5) `"TIN"` – Tax Identification Number per European Commission.  
6) Two characters by local definition, followed by `":"` (colon).  

Other initial character sequences are reserved.  
When a locally defined identity type reference is used (two characters + `":"`), the `nameRegistrationAuthorities` element of `SemanticsInformation` **shall** be present and contain at least a `uniformResourceIdentifier`. The two‑letter identity type reference **shall** be unique within the context of that URI.

#### 5.1.4 Legal person semantics identifier
When `id‑etsi‑qcs‑SemanticsId‑Legal` is included, any `organizationIdentifier` attribute **shall** contain:  
- 3‑character identity type reference;  
- 2‑character ISO 3166 country code;  
- hyphen‑minus;  
- identifier.  

The three initial characters **shall** be one of:  
1) `"VAT"` – national value added tax identification number.  
2) `"NTR"` – national trade register identifier.  
3) `"PSD"` – national authorization number of a payment service provider under Directive (EU) 2015/2366. This **shall** use the extended structure defined in ETSI TS 119 495 [3], clause 5.2.1.  
4) `"LEI"` – global Legal Entity Identifier (ISO 17442 [4]); the 2‑character country code **shall** be set to `'XG'`.  
5) Two characters by local definition, followed by `":"` (colon).  

Other initial character sequences are reserved. When `"VAT"` is used with `"EU"`, the identifier should comply with Council Directive 2006/112/EC, article 215.  
Local type constraints as in 5.1.3.

#### 5.1.5 eIDAS Natural person semantics identifier
When `id‑etsi‑qcs‑semanticsId‑eIDASNatural` is included, values of attributes in the subject field **shall** meet the content requirements of corresponding attributes defined by the eIDAS SAML attribute profile [5]:  
- `serialNumber` **shall** comply with the eIDAS `PersonIdentifier` attribute.  
- Attributes present are equivalent per Table 5.1.5‑1 (serialNumber↔PersonIdentifier, surname↔FamilyName, givenName↔FirstName, dateOfBirth↔DateOfBirth). They **shall** hold equivalent information even if format differs.

#### 5.1.6 eIDAS Legal person semantics identifier
When `id‑etsi‑qcs‑SemanticsId‑eIDASLegal` is included:  
- `organizationIdentifier` **shall** comply with the eIDAS `LegalPersonIdentifier`.  
- Attributes equivalent per Table 5.1.6‑1 (organizationIdentifier↔LegalPersonIdentifier, organizationName↔LegalName). They **shall** hold equivalent information even if format differs.

### 5.2 Certificate Extensions regarding Validity Assured Certificate
#### 5.2.1 Validity Assured General
Extensions indicate the certificate issuer ensures validity at time of use of the corresponding private key. Presence allows the relying party to decide not to check revocation status (e.g., for digital signature validation). Upper certificate path is not impacted. Only one extension is defined.

#### 5.2.2 Validity Assured - Short Term
Indicates the certificate is a “short‑term certificate”: the validity period (notBefore through notAfter) is shorter than the maximum time to process a revocation request as specified in the certificate practice statement or certificate policy.

#### 5.2.3 ASN.1 Module
The ASN.1 module **shall** import types from IETF RFC 5912 [6] as written in the import part:

```asn1
ETSIValAssuredCertMod { itu-t(0) identified-organization(4) etsi(0) id-cert-profile(194121) id-mod(0) id-mod-validity-assured(1) v1(0) }
BEGIN
-- EXPORTS All --
IMPORTS
  EXTENSION FROM PKIX-CommonTypes-2009 { iso(1) identified-organization(3) dod(6) internet(1) security(5) mechanisms(5) pkix(7) id-mod(0) id-mod-pkixCommon-02(57) }
-- Extensions
id-etsi-ext OBJECT IDENTIFIER ::= { itu-t(0) identified-organization(4) etsi(0) id-cert-profile(194121) 2 }
id-etsi-ext-valassured-ST-certs OBJECT IDENTIFIER ::= { id-etsi-ext 1 }
ext-etsi-valassured-ST-certs EXTENSION ::= { SYNTAX NULL IDENTIFIED BY id-etsi-ext-valassured-ST-certs }
END
```
