# ETSI TS 119 162-2 V1.0.1: Electronic Signatures and Infrastructures (ESI); Associated Signature Containers (ASiC); Part 2: Additional ASiC containers
**Source**: ETSI (European Telecommunications Standards Institute) | **Version**: V1.0.1 | **Date**: August 2015 | **Type**: Normative (Technical Specification)
**Original**: RTS/ESI-0019162-2-TS

## Scope (Summary)
This document specifies additional ASiC containers for use cases not covered by the baseline containers defined in ETSI TS 119 162-1 (ASiC Part 1). It covers containers that collect electronic documents, including those supported by OCF, ODF, and UCF, and describes how these container formats can be used to associate digital signatures with any data objects in the container.

## 4 ASiC additional containers
### 4.1 Introduction
This clause defines additional containers for maximizing interoperability:
- ASiC-S with time assertion (clause 4.2.1)
- ASiC-E with CAdES (clause 4.3.1)
- ASiC-E with time assertion (clause 4.3.2)

### 4.2 ASiC-S additional containers
#### 4.2.1 ASiC-S Time assertion additional container
- **Requirement**: The requirements specified in ASiC part 1 [2] for ASiC-S shall apply with the following additional restrictions:
  - a) it shall comply with the specific provisions given in ASiC part 1 [2], clause 4.3.3.1 item 2) a) and clause 4.3.3.2, item 4) a), 4) d) or 4) e)
  - b) it may contain additional elements in the META-INF folder as specified in ASiC part 1 [2], clause 4.3.3.2 item 5) a) and 5) b)
  - c) it shall support ASiC part 1 [2], clause 4.3.3.2 item 2)
  - d) if one or more ASiCArchiveManifest files are present they shall comply with ASiC part 1 [2], clause A.7 with the additional restriction that only SignedData shall be used to include certificate and revocation information.

### 4.3 ASiC-E additional containers
#### 4.3.1 ASiC-E CAdES additional container
- **Requirement**: The requirements specified in ASiC part 1 [2] for ASiC-E shall apply with the following additional restrictions:
  - a) it shall comply with the specific provisions given in ASiC part 1 [2], clause 4.4.4.1 item 1) a) and clause 4.4.4.2, item 3) a)
  - b) the CAdES signatures shall comply with CAdES [1] baseline profiles clause 6.3 B-B level, B-T level or B-LT level
  - c) it shall support ASiC part 1 [2], clause 4.4.5 item 2)

#### 4.3.2 ASiC-E Time assertion additional container
- **Requirement**: The requirements specified in ASiC part 1 [2] for ASiC-E shall apply with the following additional restrictions:
  - a) it shall comply with the specific provisions given in ASiC part 1 [2], clause 4.4.4.1 item 1) a) and clause 4.4.4.2, item 3) b) or 4) a) or 4) b)
  - b) it shall support ASiC part 1 [2], clause 4.4.5 item 2)

## Informative Annexes (Condensed)
- **Annex A (Using ASiC with existing container specifications)**: Provides recommendations for using ASiC containers compatibly with OCF, UCF, and ODF. Specifies optional files (mimetype, container.xml, manifest.xml, metadata.xml, signatures.xml) and their compliance requirements per container format. A table summarizes file presence and expected conformance for all ASiC-E containers.
- **Annex B (ASiC application to archival systems using ER)**: Describes how to extract signed data from archival systems protected by Evidence Records (ER). For a single signed data, an ASiC-S container should be created including the data and ER file. For multiple signed data, an ASiC-E container should include each data object, each ER file, and an ASiCEvidenceRecordManifest referencing them. ER hash trees may be reduced per RFC 4998/6283.
