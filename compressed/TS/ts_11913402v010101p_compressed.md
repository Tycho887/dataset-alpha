# ETSI TS 119 134‑2 V1.1.1: Electronic Signatures and Infrastructures (ESI); XAdES digital signatures – Testing Conformance and Interoperability; Part 2: Test suites for testing interoperability of XAdES baseline signatures
**Source**: ETSI | **Version**: V1.1.1 | **Date**: June 2016 | **Type**: Normative Technical Specification
**Original**: https://www.etsi.org/standards-search (DTS/ESI‑0019134‑2)

## Scope (Summary)
Defines test suites to assess interoperability between implementations claiming conformance to XAdES baseline signatures (ETSI EN 319 132‑1 [1]). Four levels are covered: B‑B, B‑T, B‑LT and B‑LTA. Test suites also cover augmentation of baseline signatures and negative test cases. PKI‑agnostic; any PKI (including EU Member States Trusted Lists) may be used.

## Definitions and Abbreviations
- **Negative test case**: a test case either for a signature that is not a XAdES baseline signature, or for a signature whose validation according to [i.4] would not result in TOTAL_PASSED.
- **CA**: Certification Authority
- **CRL**: Certificate Revocation List
- **OCSP**: Online Certificate Status Provider
- **TSA**: Time‑Stamping Authority
- **TSA1 / TSA2**: distinct Time‑Stamping Authorities (may belong to different trust hierarchies)

## Overview
The test suites are layered according to the four XAdES baseline levels defined in [1]. Table 1 lists the XML namespace prefixes used throughout the present document.

| XML Namespace URI | Prefix |
|---|---|
| http://www.w3.org/2000/09/xmldsig# | ds |
| http://uri.etsi.org/01903/v1.3.2# | xades |
| http://uri.etsi.org/01903/v1.4.1# | xadesv141 |

Mandatory properties for each level **shall** be present as specified in [1] clause 6.3.
