# ETSI TS 119 134-5 V2.1.1: Electronic Signatures and Infrastructures (ESI); XAdES digital signatures - Testing Conformance and Interoperability; Part 5: Testing Conformance of extended XAdES signatures
**Source**: ETSI | **Version**: V2.1.1 | **Date**: 2016-06 | **Type**: Normative
**Original**: RTS/ESI-0019134-5

## Scope (Summary)
Defines sets of checks (test assertions) for testing conformance of XAdES signatures against extended XAdES signatures as specified in ETSI EN 319 132-2. Only checks specific to extended XAdES are defined; common checks (for both baseline and extended) are in ETSI TS 119 134-4. The complete set of checks for an extended signature is the union of the checks in this document and those in TS 119 134-4 (groups XAdES_BB and XAdES_SCH). Does not test cryptographic validity or technical validity of the signature.

## Definitions and Abbreviations
- **XAdES**: XML Advanced Electronic Signatures
- **E-BES, E-EPES, E-T, E-C, E-X, E-X-L, E-X-Long, E-A**: Levels of extended XAdES signatures as per [2]
- **TA**: Test Assertion
- **ds:** Namespace prefix for http://www.w3.org/2000/09/xmldsig#
- **xades:** Namespace prefix for http://uri.etsi.org/01903/v1.3.2#
- **xadesv141:** Namespace prefix for http://uri.etsi.org/01903/v1.4.1#

## 4 Overview
Test assertions are grouped under "XAdES_ES" for extended signatures. The total set for testing conformance to extended XAdES signatures shall be the union of:
- Set of test assertions with code "XAdES_BB" in [4]
- Set of test assertions with code "XAdES_SCH" in [4]
- Set of test assertions defined in this document.

Tests include:
1) Structure tests derived from XML Schema (Annex A)
2) Structure tests not derivable from schema
3) Value tests (elements/attributes)
4) Interrelationship tests (URIs)
5) Computation tests (e.g., message imprints)

No tests of actual cryptographic validity.

## 5 Testing conformance to E-BES level of extended XAdES signatures

### 5.1 Introduction
This clause specifies assertions specific to XAdES-E-BES signatures (as per [2]).

### 5.2 Testing XML Signature elements and containers of XAdES qualifying properties

#### 5.2.1 Testing XML Signature elements

##### 5.2.1.1 Testing ds:Signature element
- **XADES_ES/XMLSIG/DSIG/1**: The signature has 0 or more ds:Object children. (mandatory)

##### 5.2.1.2 Testing ds:KeyInfo element
- **XADES_ES/XMLSIG/KEYINFO/1**: (Prerequisite: signature does not incorporate SigningCertificateV2) One of its ds:X509Certificate descendant elements contains the signing certificate AND this element is signed by the signature itself. (mandatory)

#### 5.2.2 Testing containers of XAdES qualifying properties and references

##### 5.2.2.1 Testing incorporation of XAdES qualifying properties to the signature
- **XAdES_ES/PROPERTIES/INCORPORATION/1**: The signature has 0 or more xades:QualifyingPropertyReferences descendant elements. (mandatory)
- **XAdES_ES/PROPERTIES/INCORPORATION/2**: The signature contains 0 or one xades:QualifyingProperties descendant elements. (mandatory)
- **XAdES_ES/PROPERTIES/INCORPORATION/3** (two occurrences, same ID?): 
  - Prerequisite: signature contains 0 or more xades:QualifyingPropertyReferences. All such elements are children of the same ds:Object element. (mandatory)
  - Different prerequisite: signature contains one xades:QualifyingProperties. That element is child of one ds:Object element. (mandatory)
- **XAdES_ES/PROPERTIES/INCORPORATION/4**: If both QualifyingPropertyReferences and QualifyingProperties exist, all are children of the same ds:Object element. (mandatory)

##### 5.2.2.2 Testing xades:QualifyingPropertiesReference
- **XAdES_ES/QPREFS/1**: The URI attribute contains a bare-name XPointer fragment AND references an external xades:QualifyingProperties element. (mandatory)
- **XAdES_ES/QPREFS/2**: In the URI attribute, the not-fragment part identifies the enclosing document, and the bare-name XPointer fragment identifies the xades:QualifyingProperties element. (mandatory)
- Plus test assertions from Annex A (see below).

##### 5.2.2.3 Testing xades:QualifyingProperties
Apply test assertions from [4] clauses 5.2.2.2.1 and A.2.1.1.

##### 5.2.2.4 Testing xades:SignedProperties
Apply assertions from [4] clause A.2.1.2 and:
- **XADES_ES/SIGPROP/1**: (Prerequisite: ds:KeyInfo does not contain signing certificate) A XAdES signature incorporates one xades:SignedProperties container element. (mandatory)
- **XADES_ES/SIGPROP/2**: (Prerequisite: ds:KeyInfo contains signing certificate but NOT signed) A XAdES signature incorporates one xades:SignedProperties container element. (mandatory)
- **XADES_ES/SIGPROP/3**: The xades:SignedProperties element is not empty. (mandatory)
- **XADES_ES/SIGPROP/4**: (Prerequisite: ds:KeyInfo does not contain signing certificate) The xades:SignedProperties contains one xades:SignedSignatureProperties container element. (mandatory)
- **XADES_ES/SIGSIGPROP/5**: (Prerequisite: ds:KeyInfo contains signing certificate but NOT signed) The xades:SignedProperties contains one xades:SignedSignatureProperties container element. (mandatory)

##### 5.2.2.5 Testing xades:SignedSignatureProperties
Apply assertions from [4] clause A.2.1.2.2 and:
- **XADES_ES/SIGSIGPROP/1**: The xades:SignedSignatureProperties element is not empty. (mandatory)
- **XADES_ES/SIGSIGPROP/2**: (Prerequisite: ds:KeyInfo does not contain signing certificate) The element contains one xades:SigningCertificateV2 qualifying property. (mandatory)
- **XADES_ES/SIGSIGPROP/3**: (Prerequisite: ds:KeyInfo contains signing certificate but NOT signed) The element contains one xades:SigningCertificateV2 qualifying property. (mandatory)

##### 5.2.2.6-5.2.2.9 Testing other property containers
These sections reference test assertions in [4] for xades:SignedDataObjectProperties, xades:UnSignedProperties, xades:UnSignedSignatureProperties, xades:UnSignedDataObjectProperties.

### 5.3 Testing XAdES qualifying properties

#### 5.3.1-5.3.9 Testing specific elements (SigningTime, SigningCertificateV2, CommitmentTypeIndication, DataObjectFormat, SignatureProductionPlaceV2, SignerRoleV2, CounterSignature, AllDataObjectsTimeStamp, IndividualDataObjectsTimeStamp)
For each, the document states that test assertions from [4] apply, plus sometimes additional assertions:
- **XADES_ES/SIGCERTV2/1**: The xades:Cert children do not contain IssuerSerialV2 child. (preferred)
- **XADES_ES/SIGCERTV2/2**: The content of IssuerSerialV2 is the base-64 encoding of one DER-encoded instance of IssuerSerial type as in [5]. (mandatory)
- **XADES_ES/SIGCERTV2/3**: The URI attribute references a place containing the X509 certificate referenced by its xades:Cert parent. (mandatory)
- CounterSignature: use same test assertions as for the main signature (see clause 5.3.7 note).
