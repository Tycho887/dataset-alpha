# ETSI TS 119 172-3 V1.1.1: Electronic Signatures and Infrastructures (ESI); Signature Policies; Part 3: ASN.1 format for signature policies
**Source**: ETSI | **Version**: V1.1.1 | **Date**: 2019-12 | **Type**: Normative Technical Specification
**Original**: https://www.etsi.org/deliver/etsi_ts/119100_119199/11917203/01.01.01_60/ts_11917203v010101p.pdf

## Scope (Summary)
This document defines an ASN.1 format for machine-readable signature policies based on the technical constraint building blocks specified in ETSI TS 119 172-1. It provides ASN.1 syntax for constraints on signature creation, validation, and augmentation; pure business-procedure applicability rules are out of scope. Mechanisms for protecting policy integrity are also out of scope.

## Definitions and Abbreviations
- **signature applicability rules**: set of rules that defines requirements for determining if a signature is fit for a particular business or legal purpose.
- **signature augmentation constraint**: criteria used when augmenting a digital signature.
- **signature augmentation policy**: set of signature augmentation constraints.
- **signature creation application**: application that creates the AdES digital signature and relies on the signature creation device.
- **signature creation constraint**: criteria used when creating a digital signature.
- **signature creation policy**: set of signature creation constraints processed by the signature creation application.
- **signature validation application**: application that validates a signature against a signature validation policy and outputs status and report.
- **signature validation constraint**: technical criteria against which a digital signature can be validated.
- **signature validation policy**: set of signature validation constraints processed by the signature validation application.
- **ASN.1**: Abstract Syntax Notation One
- **BER**: Basic Encoding Rules
- **DER**: Distinguished Encoding Rules
- **OID**: Object IDentifier
- **URI**: Uniform Resource Identifier
- **XML**: eXtensible Markup Language

## 4 ASN.1 syntax for machine processable signature policy document

### 4.1 Introduction
#### 4.1.1 Technical approach
The ASN.1 format covers only technical constraints from ETSI TS 119 172-1 building blocks. Each ASN.1 element signals whether it applies to generation, validation, augmentation, or any combination. Documents may contain components of one or more policy types.

#### 4.1.2 ASN.1 module
Annex A defines the complete ASN.1 module.

#### 4.1.3 ASN.1 encoding
##### 4.1.3.1 DER
Distinguished Encoding Rules (DER) for ASN.1 types **shall** be as specified in Recommendation ITU-T X.690 [5].

##### 4.1.3.2 BER
If Basic Encoding Rules (BER) are used for some ASN.1 types, it **shall** be as specified in Recommendation ITU-T X.690 [5].

#### 4.1.4 ASN.1 type to allow extensions: the Other type
- **Semantics**: The `Other` type **shall** contain an element of a type as defined by `OTHER.&id`. It **shall** be parameterized by a set of allowed `OTHER.&id` types (`MyOtherSet`). `MyOtherSet` **may** be extensible.
- **Syntax**: 
```asn1
OTHER ::= CLASS {
    &id OBJECT IDENTIFIER UNIQUE,
    &Value OPTIONAL
} WITH SYNTAX {
    OTHER-ID &id [OTHER-TYPE &Value]
}
Other{OTHER:MyOtherSet} ::= SEQUENCE {
    otherId OTHER.&id({MyOtherSet}),
    otherValue OTHER.&Value({MyOtherSet}{@otherId}) OPTIONAL
}
```

### 4.2 The SignaturePolicy type
- **Semantics**: As in clause 4.2.1 of ETSI TS 119 172-2 [3].
- **Syntax**: 
```asn1
SignaturePolicy ::= SEQUENCE {
    digest Digest,
    policyComponents PolicyComponents
}
```
- The `Digest` element **shall** be as specified in 4.3.2.
- The `PolicyComponents` element **shall** be as specified in 4.4.2.

### 4.3 The Digest type
- **Semantics**: As in clause 4.3.1 of ETSI TS 119 172-2 [3].
- **Syntax**: `Digest ::= OtherHashAlgAndValue` (as per ETSI EN 319 122-1 [1]).
- The `hashValue` element **shall** contain the result of applying the algorithm identified in `hashAlgorithm` on the binary encoding of the `PolicyComponents` element.

### 4.4 The PolicyComponents type
- **Semantics**: As in clause 4.4.1 of ETSI TS 119 172-2 [3].
- **Syntax**: 
```asn1
PolicyComponents ::= SEQUENCE {
    geneneralDetails GeneralDetails,
    policyRules PolicyRules
}
```
- The `GeneralDetails` element **shall** be as specified in 4.5.2.
- The `PolicyRules` element **shall** be as specified in 4.14.2.

### 4.5 The GeneralDetails type
- **Syntax**: 
```asn1
GeneralDetails ::= SEQUENCE {
    sigPolicyDetails SigPolicyDetails,
    authorityDetails [0] AuthorityDetails OPTIONAL,
    otherDetails [1] OtherDetails OPTIONAL
}
```
- Elements referenced from clauses 4.6.2, 4.7.2, 4.13.2.

### 4.6 The SigPolicyDetails type
- **Semantics**: As in clause 4.6.1 of ETSI TS 119 172-2 [3].
- **Syntax**: 
```asn1
SigPolicyDetails ::= SEQUENCE {
    policyIdentifier OBJECT IDENTIFIER,
    policyName InternationalNames,
    distributionPoints DistributionPoints OPTIONAL
}
InternationalNames ::= SEQUENCE OF MultiLangText
MultiLangText ::= SEQUENCE {
    lang PrintableString,
    text UTF8String
}
DistributionPoints ::= SEQUENCE SIZE (1..MAX) OF IA5String
```
- `policyIdentifier` **shall** contain an OID that uniquely identifies the signature policy.
- Each name in `policyName` **shall** be qualified with a language tag (IETF RFC 5646 [6], in lower case).
- `distributionPoints` **shall** contain one or more URIs.

### 4.7 The AuthorityDetails type
- **Syntax**: 
```asn1
AuthorityDetails ::= SEQUENCE {
    name [0] Name OPTIONAL,
    tradeName [1] TradeName OPTIONAL,
    postalAddresses PostalAddresses,
    electronicAddresses ElectronicAddresses,
    contactPersons [2] ContactPersons OPTIONAL
}
```
- Referenced types defined in clauses 4.8–4.12.

### 4.8 The Name type
- **Syntax**: `Name ::= InternationalNames` (as in 4.6.2).

### 4.9 The TradeName type
- **Syntax**: `TradeName ::= InternationalNames` (as in 4.6.2).

### 4.10 The PostalAddresses type
- **Syntax**: `PostalAddresses ::= SEQUENCE OF PostalAddress` (as per ETSI EN 319 122-1 [1]).

### 4.11 The ElectronicAddresses type
- **Syntax**: 
```asn1
ElectronicAddresses ::= SEQUENCE OF ElectronicAddresse
ElectronicAddresse ::= MultiLangURI
MultiLangURI ::= SEQUENCE {
    lang PrintableString,
    uri IA5String
}
```
- Each element **shall** be qualified with a language tag (IETF RFC 5646 [6], lower case).

### 4.12 The ContactPersons type
- **Syntax**: 
```asn1
ContactPersons ::= SEQUENCE OF ContactPerson
ContactPerson ::= SEQUENCE {
    name UTF8String,
    electronicAddresses ElectronicAddresses,
    phoneNumbers SEQUENCE OF PrintableString
}
```
- `phoneNumbers` each value **shall** start with '+' followed by country prefix.

### 4.13 The OtherDetails type
- **Syntax**: 
```asn1
OtherDetails ::= SEQUENCE {
    dateOfIssue GeneralizedTime,
    signingPeriod [0] SigningPeriod OPTIONAL,
    others [1] SEQUENCE SIZE (1..MAX) OF Other{{OtherDetailsOtherSet}} OPTIONAL
}
SigningPeriod ::= SEQUENCE {
    notBefore GeneralizedTime,
    notAfter GeneralizedTime OPTIONAL
}
```
- `GeneralizedTime` as per ITU-T X.680 [4].

### 4.14 The PolicyRules type
- **Syntax**: 
```asn1
PolicyRules ::= SEQUENCE OF PolicyRuleWithScope
PolicyRuleWithScope ::= SEQUENCE {
    rule SigPolicyRule,
    scope SigPolicyScope OPTIONAL
}
SigPolicyRule ::= CHOICE {
    commitmentRules [0] CommitmentRules,
    basicRule [1] BasicRule
}
BasicRule ::= CHOICE { ... }  (see complete definition in annex A)
SigPolicyScope ::= ENUMERATED { generation (0), validation (1), augmentation (2) }
```
- Each `SigPolicyScope` value indicates the phase to which the rule applies.

### 4.15 The CommitmentRules type
- **Syntax**: 
```asn1
CommitmentRules ::= SEQUENCE OF CommitmentRule
CommitmentRule ::= SEQUENCE {
    commitments SEQUENCE OF Commitment,
    matchingIndicator MatchingIndicator,
    basicRules SEQUENCE OF BasicRule
}
Commitment ::= SEQUENCE {
    commitmentIdentifier CommitmentTypeIdentifier,
    details SEQUENCE SIZE (1..MAX) OF MultiLangString OPTIONAL
}
MatchingIndicator ::= ENUMERATED { all (0), none (1), atLeastOne (2) }
```
- Each `Commitment` **shall** contain an identifier; `details` **may** provide additional text qualified by language.

### 4.16 The DataToBeSignedRules type
- **Syntax**: 
```asn1
DataToBeSignedRules ::= SEQUENCE OF DataToBeSignedRule
DataToBeSignedRule ::= SEQUENCE {
    anyOfMimeType [0] SEQUENCE SIZE (1..MAX) OF UTF8String OPTIONAL,
    noneOfMimeType [1] SEQUENCE SIZE (1..MAX) OF UTF8String OPTIONAL
}
```
- `DataToBeSignedRule` **shall not** be empty.

### 4.17 The SigToDTBSRelationRules type
- **Syntax**: 
```asn1
SigToDTBSRelationRules ::= SEQUENCE {
    dTBSCardinality DTBSCardinality,
    sigDTBSRelativePosition [0] SigDTBSRelativePosition OPTIONAL,
    sigFormatsAndLevels [1] SigFormatsAndLevels OPTIONAL
}
```

### 4.18 The DTBSCardinality type
- **Syntax**: 
```asn1
DTBSCardinality ::= SEQUENCE {
    maxDTBSNumber [0] MaxDTBSNumber OPTIONAL,
    minDTBSNumber [1] MinDTBSNumber OPTIONAL
}
MaxDTBSNumber ::= SEQUENCE { dTBSNumber INTEGER, maxValueQualifier MaxValueQualifier }
MaxValueQualifier ::= ENUMERATED { lessThan (0), lessOrEqualTo (1), equal (2) }
MinDTBSNumber ::= SEQUENCE { dTBSNumber INTEGER, minValueQualifier MinValueQualifier }
MinValueQualifier ::= ENUMERATED { higherThan (0), higherOrEqualTo (1), equal (2) }
```

### 4.19 The SigDTBSRelativePosition type
- **Syntax**: 
```asn1
SigDTBSRelativePosition ::= SEQUENCE OF SigDTBSRelativePositionValue
SigDTBSRelativePositionValue ::= ENUMERATED {
    envelopingSig (0), envelopedSig (1), detachedSig (2), aSiC (3)
}
```
- If `aSiC` is present, no other value **shall** be present. Otherwise the signature **shall** be placed as indicated by each value for one or more signed data objects.

### 4.20 The SigFormatsAndLevels type
- **Syntax**: 
```asn1
SigFormatsAndLevels ::= SEQUENCE {
    sigFormats [0] SEQUENCE SIZE (1..MAX) OF IA5String OPTIONAL,
    sigLevels [1] SEQUENCE SIZE (1..MAX) OF IA5String OPTIONAL
}
```
- Instances **shall not** be empty. Absence of `sigFormats` means any format allowed. Absence of `sigLevels` means any level allowed.

### 4.21 The AugmentationRules type
- **Syntax**: 
```asn1
AugmentationRules ::= SEQUENCE {
    previousValidationRequired BOOLEAN,
    levelID IA5String,
    augQualifier AugmentationQualifier
}
AugmentationQualifier ::= ENUMERATED { thisLevel (0), minLevel (1), maxLevel (2) }
```
- `levelID` **shall** contain a URI describing the augmentation level.

### 4.22 Types for defining constraints on certificates' trust and revocation status

#### 4.22.2 The TrustAnchors type
- **Syntax**: 
```asn1
TrustAnchors ::= SEQUENCE OF TrustAnchor
TrustAnchor ::= CHOICE {
    certificate CertAndReliableTime,
    tAInTrustedList TAsInTrustedList
}
CertAndReliableTime ::= SEQUENCE {
    cert Certificate,
    reliableUntil GeneralizedTime OPTIONAL
}
TAsInTrustedList ::= SEQUENCE {
    uri IA5String,
    serviceTypes [0] SEQUENCE SIZE (1..MAX) OF IA5String OPTIONAL,
    serviceStatuses [1] SEQUENCE SIZE (1..MAX) OF IA5String OPTIONAL
}
```
- `Certificate` as per IETF RFC 5912 [7]. `uri` **shall** reference a Trusted List per ETSI TS 119 612 [2].

#### 4.22.3 The NameConstraints type
- **Syntax**: 
```asn1
NameConstraints ::= SEQUENCE {
    permittedSubtrees [0] GeneralSubtrees OPTIONAL,
    excludedSubtrees [1] GeneralSubtrees OPTIONAL
}
GeneralSubtrees ::= SEQUENCE SIZE (1..MAX) OF GeneralSubtree
GeneralSubtree ::= SEQUENCE {
    base GeneralName,
    minimum [0] BaseDistance DEFAULT 0,
    maximum [1] BaseDistance OPTIONAL
}
```

#### 4.22.4 The PolicyConstraints type
- **Syntax**: 
```asn1
PolicyConstraints ::= SEQUENCE {
    requireExplicitPolicy [0] SkipCerts OPTIONAL,
    inhibitPolicyMapping [1] SkipCerts OPTIONAL
}
SkipCerts ::= INTEGER (0..MAX)
```

#### 4.22.5 The CertificateTrustTrees type
- **Syntax**: 
```asn1
CertificateTrustTrees ::= SEQUENCE OF CertificateTrustPoint
CertificateTrustPoint ::= SEQUENCE {
    trustAnchors TrustAnchors,
    pathLenConstraint [0] PathLenConstraint OPTIONAL,
    acceptablePolicySet [1] AcceptablePolicySet OPTIONAL,
    nameConstraints [2] NameConstraints OPTIONAL,
    policyConstraints [3] PolicyConstraints OPTIONAL,
    userCertPath [4] UserCertPath OPTIONAL
}
```

### 4.23 Types for defining constraints on certificates' revocation status

#### 4.23.2 The CertificateRevReq type
- **Syntax**: 
```asn1
CertificateRevReq ::= SEQUENCE {
    endCertRevReq EnuRevReq,
    caCerts EnuRevReq
}
EnuRevReq ::= ENUMERATED { clrCheck, ocspCheck, bothCheck, eitherCheck, noCheck, other }
```

#### 4.23.3 The CertificateRevTrust type
- **Syntax**: 
```asn1
CertificateRevTrust ::= SEQUENCE {
    certificateRevReq CertificateRevReq,
    freshness [0] Freshness OPTIONAL,
    sigCertIssuedByCAKeepsExpiredRevokedCertsInfo [1] Duration OPTIONAL
}
Freshness ::= CHOICE {
    maxDifferenceRevocationAndValidation [0] Duration,
    timeAfterSignature [1] Duration
}
Duration ::= PrintableString
```
- `timeAfterSignature` **shall** be the time after best-signature time as defined in ETSI TS 119 102-1 [i.1].
- If `sigCertIssuedByCAKeepsExpiredRevokedCertsInfo` is present, the CA **shall** keep revocation info for expired certificates beyond the indicated duration (XML Schema duration format).

### 4.24 The SigningCertRules type
- **Syntax**: 
```asn1
SigningCertRules ::= SEQUENCE {
    signingCertTrustConditions SigningCertTrustConditions,
    mandatedSigningCertInfo MandatedSigningCertInfo OPTIONAL
}
MandatedSigningCertInfo ::= ENUMERATED { signingCertOnly (0), fullPath (1) }
SigningCertTrustConditions ::= SEQUENCE {
    signerTrustTrees CertificateTrustTrees,
    signerRevTrust CertificateRevTrust
}
```

### 4.25 The TimeEvidencesRules type
- **Syntax**: 
```asn1
TimeEvidencesRules ::= SEQUENCE OF RulesForSetOfEvidences
RulesForSetOfEvidences ::= SEQUENCE {
    evidenceIdentifiers SEQUENCE OF IA5String,
    levelOfAssurance IA5String,
    timeStampTrustCondition TimestampTrustCondition OPTIONAL
}
TimestampTrustCondition ::= SEQUENCE { ... }  (see detailed definition in annex A)
```

### 4.26 The SignerAttributeConstraints type
- **Syntax**: 
```asn1
SignerAttributeConstraints ::= SEQUENCE {
    noSignerAttributesAllowed BOOLEAN,
    constraintsOnOneSetOfAttributes SEQUENCE OF AttributeSetConstraints
}
AttributeSetConstraints ::= SEQUENCE { ... }
HowCertAttribute ::= ENUMERATED { claimedAttribute, certifiedAttribtes, signedAssertions, any }
```

### 4.27 The QualifyingAttributesRules type
- **Syntax**: 
```asn1
QualifyingAttributesRules ::= SEQUENCE OF LevelAttributesRules
LevelAttributesRules ::= SEQUENCE {
    levelIdentifier [0] IA5String,
    signedAttributes [1] SignatureAttributes OPTIONAL,
    unsignedAttributes [2] SignatureAttributes OPTIONAL
}
SignatureAttributes ::= SEQUENCE OF CHOICE { choice [0] SEQUENCE OF SignatureAttribute, sigAttr [1] SignatureAttribute }
SignatureAttribute ::= SEQUENCE { identifier OBJECT IDENTIFIER, mandatory BOOLEAN }
```

### 4.28 The SCDLoARules type
- **Syntax**: `SCDLoARules ::= IA5String`
- The value **shall** be a URI indicating the Level of Assurance of the signature creation device.

### 4.29 The CryptoSuitesRules type
- **Syntax**: 
```asn1
CryptoSuitesRules ::= SEQUENCE OF AlgConstraints
AlgConstraints ::= SEQUENCE {
    algID OBJECT IDENTIFIER,
    usages SEQUENCE SIZE (1..MAX) OF IA5String,
    minKeyLength [0] INTEGER OPTIONAL,
    minHashLength [1] INTEGER OPTIONAL,
    other Other{{AlgConstraintsOtherSet}} OPTIONAL
}
```
