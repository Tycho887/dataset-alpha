# ETSI TS 119 124-4 V1.1.1 (2016-06): CAdES digital signatures – Testing Conformance and Interoperability; Part 4: Testing conformance of CAdES baseline signatures
**Source**: ETSI TC ESI | **Version**: V1.1.1 | **Date**: 2016-06 | **Type**: Normative Technical Specification
**Original**: [ETSI TS 119 124-4](http://www.etsi.org/standards-search)

## Scope (Summary)
Defines the set of checks for testing conformance of CAdES signatures against CAdES baseline signatures as specified in ETSI EN 319 122-1 [1]. Does not specify checks for technical validity of signatures; only structural and element constraints defined by CAdES [1]. Agnostic to development technology and implementation strategies.

## Normative References
- [1] ETSI EN 319 122-1: "CAdES digital signatures; Part 1: Building blocks and CAdES baseline signatures".
- [2] ETSI EN 319 122-2: "CAdES digital signatures; Part 2: Extended CAdES signatures".
- [3] IETF RFC 5652 (09-2009): "Cryptographic Message Syntax (CMS)".

## Definitions and Abbreviations
- **CAdES_BS**: Requirements from clauses 5 and 6 of ETSI EN 319 122-1 [1] specific to CAdES baseline signatures.
- **CAdES_BB**: Requirements from clauses 4, 5 and Annex A of ETSI EN 319 122-1 [1] common to both baseline and extended signatures.
- **Test Assertion (TA)**: Defined per OASIS Test Assertions Guidelines [i.3], with unique identifier, normative source, target, predicate, prescription level (mandatory/recommended/permitted), and tag.
- Other definitions and abbreviations as given in ETSI TR 119 001 [i.2].

## 4 Overview
Requirements classified into:
- **CAdES_BS** (CAdES baseline signatures specific)
- **CAdES_BB** (CAdES building blocks common to baseline and extended)

Tests cover: signature structure, element/attribute values, interrelationships, computations (e.g., message imprints). No tests on cryptographic validity. Test assertions identifiers start with `CAdES_BS_` or `CAdES_BB_`.

## 5 Testing conformance to CAdES-B-B signatures

### 5.1 Introduction
Clause 5.2: assertions for CMS Signature elements. Clause 5.3: assertions for basic attributes.

### 5.2 Testing CMS Signature elements

#### 5.2.1 Testing algorithm requirements
- **CAdES_BS/ALG/1**: For new signatures, applications shall not use MD5 as digest algorithm. (mandatory)

#### 5.2.2 Testing content-type
- **CAdES_BB/CTY/1**: Applications shall include ContentType attribute in CMS signature. (mandatory)
- **CAdES_BS/CTY/1**: Applications shall set the value id-data in ContentType. (mandatory)

#### 5.2.3 Testing message-digest
- **CAdES_BB/MD/1**: Applications shall include message-digest attribute. (mandatory)

#### 5.2.4 Testing CMSversion
- **CAdES_BB/CMSV**: Applications shall include SignedData.CMSversion. (mandatory)

#### 5.2.5 Testing DigestAlgorithmIdentifiers
- **CAdES_BB/DAI**: Applications shall include SignedData.DigestAlgorithmIdentifiers. (mandatory)

#### 5.2.6 Testing EncapsulatedContentInfo
- **CAdES_BB/ECI**: Applications shall include SignedData.EncapsulatedContentInfo. (mandatory)

#### 5.2.7 Testing SignedData.certificates
- **CAdES_BB/SDC/1**: Applications should include all certificates needed for path building. (recommended)
- **CAdES_BB/SDC/2**: Applications should avoid duplication of certificate values. (recommended)
- **CAdES_BS/SDC/1**: Applications shall include the signing certificate. (mandatory)

#### 5.2.8 Testing SignedData.crls
- **CAdES_BB/SDCRL/1**: Applications may include full set of CRL values. (permitted)
- **CAdES_BB/SDCRL/2**: Applications may include full set of OCSP responses. (permitted)
- **CAdES_BB/SDCRL/3**: Applications should avoid duplication of revocation values. (recommended)

#### 5.2.9 Testing SignerInfos
- **CAdES_BB/SI/1**: Applications shall include one or more SignerInfos. (mandatory)

### 5.3 Testing basic attributes for CAdES signatures

#### 5.3.1 Testing signing-time
- **CAdES_BB/STI/1**: SigningTime value encoded as UTCTime for dates 1950-2049 inclusive, else GeneralizedTime. (mandatory)
- **CAdES_BS/STI/1**: Applications shall include SigningTime attribute. (mandatory)

#### 5.3.2 Testing Enhanced Security Services (ESS)
- **CAdES_BB/ESS/1**: Applications shall include ESS signing-certificate or signing-certificate-v2 in signedAttrs. (mandatory)
- **CAdES_BB/ESS/2**: If SHA-1 used, shall include ESS-signing-certificate. (mandatory)
- **CAdES_BB/ESS/3**: If other hash than SHA-1 used, shall include ESS-signing-certificate-v2. (mandatory)
- **CAdES_BB/ESS/4**: Applications should not include issuerSerial field in ESS-signing-certificate. (recommended)
- **CAdES_BS/ESS/1**: Applications should include ESS signing-certificate-v2. (recommended)

#### 5.3.3 Testing countersignature
- **CAdES_BB/CS/1**: CMS signature may include countersignature attribute. (permitted)

#### 5.3.4 Testing content-reference
- **CAdES_BB/CR/1**: Applications may include content-reference. (permitted)

#### 5.3.5 Testing content-identifier
- **CAdES_BB/CI/1**: Applications may include content-identifier. (permitted)

#### 5.3.6 Testing content-hints
- **CAdES_BB/CH/1**: Applications may include content-hints. (permitted)

#### 5.3.7 Testing commitment-type-indication
- **CAdES_BB/CTI/1**: Applications may include commitment-type-indication. (permitted)

#### 5.3.8 Testing signer-location
- **CAdES_BB/SL/1**: Applications may include signer-location. (permitted)

#### 5.3.9 Testing signer-attributes-v2
- **CAdES_BB/SA/1**: Applications may include signer-attributes-v2. (permitted)

#### 5.3.10 Testing content-time-stamp
- **CAdES_BB/CTS/1**: Applications may include content-time-stamp. (permitted)

#### 5.3.11 Testing mime-type
- **CAdES_BB/MT/1**: Applications may include mime-type. (permitted)

#### 5.3.12 Testing signature-policy-identifier
- **CAdES_BB/SPID/1**: Applications may include signature-policy-identifier. (permitted)

#### 5.3.13 Testing signature-policy-store
- **CAdES_BB/SPS/1**: If signature-policy-identifier present, applications may include signature-policy-store. (permitted)

## 6 Testing conformance to CAdES-B-T signatures

### 6.1 General
CAdES-B-T signatures shall fulfill requirements of clause 5 and all requirements of clause 6.2.

### 6.2 Testing CAdES attributes

#### 6.2.1 Testing SignatureTimeStamp
- **CAdES_BB/STS/1**: For new signatures, applications shall include signature-time-stamp attribute. Target: CAdES-B-T or extended signatures. (mandatory)

## 7 Testing conformance to CAdES-B-LT signatures

### 7.1 General
CAdES-B-LT signatures shall fulfill requirements of clauses 5, 6, and all of clause 7.2.

### 7.2 Testing CAdES attributes

#### 7.2.1 Testing Certificate and Revocation references and values
- **CAdES_BS/CCR/1**: Applications shall not include complete-certificate-references. (mandatory)
- **CAdES_BS/CRR/1**: Applications shall not include complete-revocation-references. (mandatory)
- **CAdES_BS/ACR/1**: Applications shall not include attribute-certificate-references. (mandatory)
- **CAdES_BS/ARR/1**: Applications shall not include attribute-revocation-references. (mandatory)
- **CAdES_BS/CV/1**: Applications shall not include certificate-values. (mandatory)
- **CAdES_BS/RV/1**: Applications shall not include revocation-values. (mandatory)

#### 7.2.2 Testing time-stamp attributes
- **CAdES_BS/ESCTS/1**: Applications shall not include CAdES-C-time-stamp. (mandatory)
- **CAdES_BS/TSCCR/1**: Applications shall not include time-stamped-certs-crls-references. (mandatory)

#### 7.2.3 Testing SignedData.certificates attribute
- **CAdES_BS/SDC/2**: Applications shall include all certificates needed for path building. (mandatory)

#### 7.2.4 Testing revocation values
- **CAdES_BS/SDCRL/1**: Applications shall include full set of revocation data (CRL or OCSP) used in validation. (mandatory)

## 8 Testing conformance to CAdES-B-LTA signatures

### 8.1 General
CAdES-B-LTA signatures shall fulfill requirements of clauses 5, 6, 7, and all of clause 8.2.

### 8.2 Testing CAdES attributes

#### 8.2.1 Testing ArchiveTimeStampV3
- **CAdES_BB/ATSV3/1**: Applications shall include archive-time-stamp-v3 attribute. Target: CAdES-B-LTA or CAdES-E-A. (mandatory)
- **CAdES_BB/ATSHI/1**: If archive-time-stamp-v3 present, applications shall include ats-hash-index-v3 attribute within it. (mandatory)

#### 8.2.2 Testing time-stamp attributes
- **CAdES_BB/ATS/1**: Applications shall not include archive-time-stamp (OID 1.2.840.113549.1.9.16.2.27). (mandatory)
- **CAdES_BB/ATSV2/1**: Applications shall not include archive-time-stampV2 (OID 1.2.840.113549.1.9.16.2.48). (mandatory)
- **CAdES_BB/LTV/1**: Applications shall not include long-term-validation attribute. (mandatory)

## Annex A (normative): Test assertions derived from attributes definition

### A.1 General
This annex specifies test assertions focused on ASN.1 structure of attributes defined by CAdES [1] and CMS [3]. All assertions are mandatory for applications claiming conformance to CAdES signatures as specified in [1] or [2].

### A.2 Testing CMS defined attributes
- **CAdES_ASN/CTY/1 to 4**: OID and type definitions for content-type, data, signed-data.
- **CAdES_ASN/MD/1-2**: OID and OCTET STRING for message-digest.
- **CAdES_ASN/CMSV/1**: CMSVersion ASN.1 INTEGER {1,2,3,4,5}.
- **CAdES_ASN/DAI/1**: DigestAlgorithmIdentifiers SET OF SEQUENCE with OID.
- **CAdES_ASN/SAI/1**: SignatureAlgorithmIdentifiers similar.
- **CAdES_ASN/ECI/1**: EncapsulatedContentInfo SEQUENCE with OID and optional OCTET STRING.
- **CAdES_ASN/SDC/1**: SignedData.certificates SET OF CHOICES.
- **CAdES_ASN/SDCRL/1**: SignedData.crls SET OF CHOICES.
- **CAdES_ASN/SI/1**: SignerInfo SEQUENCE with specified fields.
- **CAdES_ASN/AI/1**: AlgorithmIdentifier SEQUENCE with OID.

### A.3 Testing basic attributes for CAdES signatures
- **signing-time**: OID, CHOICE UTCTime/GeneralizedTime, encoding rules (CAdES_ASN/STI/1-3).
- **ESS signing-certificate**: OID and ASN.1 structure (SEQUENCE of ESSCertID, etc.) (CAdES_ASN/ESS/1-5).
- **ESS signing-certificate-v2**: OID and structure (SEQUENCE of ESSCertIDv2) (CAdES_ASN/ESSv2/1-3).
- **countersignature**: OID, value of type SignerInfo (CAdES_ASN/CS/1-2).
- **content-reference**: OID, value SEQUENCE {ContentType, ContentIdentifier, OCTET STRING} (CAdES_ASN/CR/1-2).
- **content-identifier**: OID, value OCTET STRING (CAdES_ASN/CI/1-2).
- **content-hints**: OID, value SEQUENCE {UTF8String, ContentType} (CAdES_ASN/CH/1-2).
- **commitment-type-indication**: OID, value SEQUENCE of CommitmentTypeIdentifier, etc. (CAdES_ASN/CTI/1-4).
- **signer-location**: OID, value SEQUENCE with optional countryName, localityName, postalAddress; at least one shall be present. (CAdES_ASN/SL/1-4).
- **signer-attributes-v2**: OID, value SEQUENCE with optional claimedAttributes, certifiedAttributesV2, signedAssertions; detailed ASN.1 structure. (CAdES_ASN/SA/1-7).
- **content-time-stamp**: OID, value TimeStampToken (CAdES_ASN/CTS/1-2).
- **mime-type**: OID, value UTF8String (CAdES_ASN/MT/1-2).
- **signature-policy-identifier**: OID, value CHOICE of SignaturePolicyId, etc.; detailed ASN.1 (CAdES_ASN/SPI/1-5).
- **signature-policy-store**: OID, value SEQUENCE of SPDocSpecification, SignaturePolicyDocument (OCTET STRING or IA5String) (CAdES_ASN/SPS/1-3).
- **signature-time-stamp**: OID, value TimeStampToken (CAdES_ASN/STS/1-2).
- **complete-certificate-references**: OID, value SEQUENCE of {OtherHash, IssuerSerial OPTIONAL} (CAdES_ASN/CCR/1-5).
- **complete-revocation-references**: OID, value SEQUENCE with crlids, ocspids, otherRev options; detailed ASN.1 (CAdES_ASN/CRR/1-10).
- **certificate-values**: OID, value SEQUENCE of Certificate (CAdES_ASN/CV/1-2).
- **revocation-values**: OID, value SEQUENCE with crlVals, ocspVals, otherRevVals (CAdES_ASN/RV/1-2).
- **CAdES-C-time-stamp**: OID, value TimeStampToken (CAdES_ASN/ESCTS/1-2).
- **time-stamped-certs-crls-references**: OID, value TimeStampToken (CAdES_ASN/TSCCR/1-2).
- **ArchiveTimeStampV3**: OID, value TimeStampToken (CAdES_ASN/ATSV3/1-2).
- **ats-hash-index-v3**: OID, value SEQUENCE of AlgorithmIdentifier and three SEQUENCE OF OCTET STRING (CAdES_ASN/ATSHI/1-2).
- **long-term-validation**: OID, value SEQUENCE with GeneralizedTime, optional timeStamp/evidenceRecord, extraCertificates, extraRevocation (CAdES_ASN/LTV/1-2).
- **ArchiveTimeStampV2**: OID, value TimeStampToken (CAdES_ASN/ATSV2/1-2).
