# ETSI TS 119 524-1: Electronic Signatures and Infrastructures (ESI); Testing Conformance and Interoperability of Electronic Registered Delivery Services; Part 1: Testing conformance
**Source**: ETSI | **Version**: V1.2.1 | **Date**: 2023-12 | **Type**: Normative (Technical Specification)
**Original**: https://www.etsi.org/deliver/etsi_ts/119500_119599/11952401/01.02.01_60/ts_11952401v010201p.pdf

## Scope (Summary)
Defines checks for testing conformance of Electronic Registered Delivery Service (ERDS) provision against ETSI EN 319 522-3 (formats), ETSI EN 319 522-4-1/4-2 (bindings), and ETSI EN 319 522-1 (framework). Covers structure/contents of ERDS evidence and metadata, AS4 message construction, digital signatures, Common Services Interface aspects, and service provision (evidence generation and security). Does **not** cover conformance to AS4, BDXL, SMP, or digital signature specifications outside the EN 319 522 series.

## 4 Overview
Tests categorized as: (1) XML Schema derived, (2) non-Schema derived, (3) element/attribute values, (4) interrelationships. Each TA includes: unique identifier, normative source, target, prerequisite, predicate, prescription level (mandatory/preferred/permitted), tag. Prefixes used: `erds` (default namespace `http://uri.etsi.org/19522/v1#`), `xs`, `ds`, `saml`.

## 8 Test assertions for ERDS AS4-profile messages
### 8.1 Generic requirements
- **ERDS/AS4_PROF/TA_01**: ERDS conforms to AS4 ebHandler Conformance Clause (section 6.1 of [15]). (mandatory)
- **TA_02**: Only push message exchange pattern. (mandatory)
- **TA_03**: ERD messages packaged in User Messages. (mandatory)
- **TA_04**: User content, relay metadata, evidence are ebMS payloads as SOAP attachments; SOAP Body not used. (mandatory)
- **TA_05**: SOAP Body not used. (mandatory)
- **TA_06-07**: Compression (AS4 Compression Feature or HTTP gzip) may be used; if used, must be one of those. (optional)
- **TA_08**: Each payload has PartInfo/Property with name `http://uri.etsi.org/19522/v1#as4binding/PayloadType`. (mandatory)
- **TA_09-11**: Value of that property: "UserContent" for user content, "ERDSRelayMetadata" for metadata, "ERDSEvidence" for evidence. (mandatory)
- **TA_12**: AS4 message always includes at least one payload containing metadata per ETSI EN 319 522-3 [3]. (mandatory)
- **TA_13-14**: PMode.Initiator and PMode.Responder include identifiers of sending and receiving ERDSs. (mandatory)
- **TA_15**: Roles: `http://uri.etsi.org/19522/v1#/as4binding/Roles/ERDS`. (mandatory)
- **TA_16**: Service set to `http://uri.etsi.org/19522/v1#/as4binding/Relay`. (mandatory)
- **TA_17**: Service type not used. (mandatory)
- **TA_18**: PMode[1].Security.SendReceipt and .NonRepudiation = true. (mandatory)
- **TA_19**: ReplyPattern and Report.AsResponse = true. (mandatory)
- **TA_20**: AS4 Reception Awareness Feature should be used. (recommended)
- **TA_21**: Duplicate elimination function should be used. (recommended)

### 8.2 Signing and encryption
- **TA_1**: All AS4 messages signed and encrypted by sending ERDS. (mandatory)
- **TA_2**: Hash algorithm per ETSI TS 119 312 [16]. (mandatory)
- **TA_3-4**: Binary Security Token reference (per [i.1]) should be used; if used, must reference X509v3 signing certificate. (recommended/mandatory)
- **TA_5**: Signature algorithm per TS 119 312. (mandatory)
- **TA_6**: Encryption algorithm AES-GCM128. (mandatory)
- **TA_7**: Encryption key transport algorithm RSA-OAEP. (mandatory)
- **TA_8**: Encryption key mask algorithm MGF1 with SHA256. (mandatory)

### 8.3 Binding of ERD Dispatch
- **TA_01**: Action: `http://uri.etsi.org/19522/v1#/as4binding/Actions/ERDdispatch`. (mandatory)
- **TA_02**: One payload per //UserContentInfo//PartInfo element. (mandatory)
- **TA_03-04**: Part property `.../UserContentPartId` with value from //PartInfo/Identifier. (mandatory)
- **TA_05-06**: May include one or more payloads containing ERDS evidence; for each, part property `.../PayloadType` should be "ERDSEvidence". (optional/recommended)

### 8.4 Binding of ERDS Receipt
- **TA_01**: Action: `.../Actions/ERDSreceipt`. (mandatory)
- **TA_02**: Message includes one or more payloads containing ERDS evidence. (mandatory)

### 8.5 Binding of ERD ServiceInfo
- **TA_01**: Action: `.../Actions/ERDserviceInfo`. (mandatory)
- **TA_02**: No further payloads beyond metadata. (mandatory)

### 8.6 Binding of ERD Payload
- **TA_01**: Action: `.../Actions/ERDpayload`. (mandatory)
- **TA_02**: One payload per //UserContentInfo//PartInfo. (mandatory)
- **TA_03-05**: Each payload has two part properties: PayloadType="UserContentPart" and UserContentPartId = //PartInfo/Identifier. (mandatory)
- **TA_06**: No other payloads beyond metadata and user content parts. (mandatory)
