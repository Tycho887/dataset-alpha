# ETSI TS 119 132-1: XAdES Digital Signatures – Part 1: Building Blocks and XAdES Baseline Signatures
**Source**: ETSI/ESI | **Version**: V1.0.1 | **Date**: 2015-07 | **Type**: Normative Technical Specification  
**Original**: RTS/ESI-0019132-1-TS

## Scope (Summary)
Specifies XAdES digital signatures building on XMLDSIG by incorporating signed/unsigned qualifying properties. Defines XML Schema definitions, mechanisms for incorporation, and four levels of XAdES baseline signatures (B‑B, B‑T, B‑LT, B‑LTA) for long‑term validity. Out of scope: creation/validation procedures (see ETSI TS 119 102‑1).

## Definitions and Abbreviations
- **attribute certificate**: data structure digitally signed by an attribute authority binding attributes with holder identification.
- **certificate revocation list (CRL)**: signed list of revoked certificates.
- **data object**: binary/octet data operated on by application (per XMLDSIG).
- **digital signature**: cryptographic transformation for source/integrity proof and forgery protection.
- **digital signature value**: result of cryptographic transformation.
- **electronic time‑stamp**: data binding other data to a particular time (per RFC 3161/5816).
- **legacy XAdES signature**: signature per ETSI TS 101 903 or TS 103 171.
- **message imprint**: digest value of data to be time‑stamped.
- **trust anchor**: entity trusted for certificate path validation.
- **XAdES signature**: signature satisfying this document or TS 119 132‑2.

Abbreviations: ASN.1, CA, CRL, DER, OCSP, OID, PKI, TSA, URI, URL, URN, UTC, XMLDSIG.

## 4 General Syntax
### 4.1 General Requirements
XAdES signatures **shall** build on XMLDSIG [1] by incorporating signed/unsigned qualifying properties as instances of XML Schema types [2][3].

### 4.2 XML Namespaces
- `http://uri.etsi.org/01903/v1.3.2#` (prefix `ds` for `http://www.w3.org/2000/09/xmldsig#`, `xsd` for `http://www.w3.org/2001/XMLSchema`)
- `http://uri.etsi.org/01903/v1.4.1#`
Two XML Schema files accompany the document: `XAdES01903v132-201506.xsd` and `XAdES01903v141-201506.xsd`. In case of discrepancies, the XML Schema files **shall** take precedence.

### 4.3 QualifyingProperties Container
- **QualifyingProperties** element **shall** contain all qualifying information. **Target** attribute **shall** reference the `Id` of the `ds:Signature`. If empty, **shall not** be incorporated.

#### 4.3.2 SignedProperties
- Contains signed qualifying properties. One `ds:Reference` **shall** point to it with `Type="http://uri.etsi.org/01903#SignedProperties"`. **Shall not** be empty.

#### 4.3.3 UnsignedProperties
- Contains unsigned qualifying properties. **Shall not** be empty.

#### 4.3.4 SignedSignatureProperties
- Contains signed qualifying properties qualifying the signature. Deprecated properties `SigningCertificate`, `SignatureProductionPlace`, `SignerRole` **shall not** be used; replaced by V2 versions.

#### 4.3.5 SignedDataObjectProperties
- Contains signed qualifying properties qualifying signed data objects. **Shall not** be empty.

#### 4.3.6 UnsignedSignatureProperties
- Contains unsigned qualifying properties qualifying the signature. Deprecated properties (`CompleteCertificateRefs`, `SigAndRefsTimeStamp`, etc.) **shall not** be used; replaced by V2 versions. **Shall not** be empty.

#### 4.3.7 UnsignedDataObjectProperties
- Reserved for future use; **shall not** be empty if used.

### 4.4 Incorporating Qualifying Properties
- Use `ds:Object` for direct or indirect (via `QualifyingPropertiesReference`) incorporation.
- Direct: `QualifyingProperties` as child of `ds:Object`.
- Indirect: `QualifyingPropertiesReference` referencing external `QualifyingProperties`.
- Only one `QualifyingProperties` per `ds:Object`; all signed properties in a single `QualifyingProperties`.

#### 4.4.2 Signing Properties
- A `ds:Reference` **shall** reference the `SignedProperties` element with `Type="http://uri.etsi.org/01903#SignedProperties"`.

#### 4.4.3 QualifyingPropertiesReference
- **URI** attribute **shall** contain bare‑name XPointer fragment referencing external `QualifyingProperties`.

### 4.5 Managing Canonicalization of XML Nodesets
- When generating new XAdES signatures or augmenting legacy signatures, all qualifying properties that provide optional canonicalization algorithm **shall** include the algorithm identifier.

## 5 Qualifying Properties Semantics and Syntax
### 5.1 Auxiliary Syntax
#### 5.1.1 AnyType
- Allows arbitrary XML elements, text, and attributes.

#### 5.1.2 ObjectIdentifierType
- **Identifier** **shall** contain permanent URI or OID (encoded as URN or URI). OID as URN: `Qualifier="OIDAsURN"`, OID as URI: `Qualifier="OIDAsURI"`. **Description** and **DocumentationReferences** optional.

#### 5.1.3 EncapsulatedPKIDataType
- Base‑64 encoded PKI object. **Encoding** attribute values: `#DER`, `#BER`, `#CER`, `#PER`, `#XER`. If absent, DER assumed.

#### 5.1.4 Types for Electronic Time‑Stamps
##### 5.1.4.3 GenericTimeStampType (abstract)
- Contains `Include` or `ReferenceInfo`, optional `CanonicalizationMethod`, and one or more `EncapsulatedTimeStamp` or `XMLTimeStamp`.

##### 5.1.4.4 XAdESTimeStampType
- For time‑stamps on XAdES components. Explicit mechanism via `Include` referencing data objects; implicit mechanism for certain properties (e.g., `AllDataObjectsTimeStamp`).

##### 5.1.4.5 OtherTimeStampType
- For time‑stamps on external data objects; uses `ReferenceInfo` elements.

### 5.2 Basic Qualifying Properties
#### 5.2.1 SigningTime
- Signed. Value: claimed UTC signing time.

#### 5.2.2 SigningCertificateV2
- Signed. Contains reference to signing certificate (digest + issuerSerialV2) and optionally path certificates. First reference **shall** be signing certificate. `IssuerSerialV2` optional; `URI` attribute optional.

#### 5.2.3 CommitmentTypeIndication
- Signed. Indicates signer’s commitment for all or subset of signed data objects. `CommitmentTypeId` via URI (not OID). `ObjectReference` references `ds:Reference`; or `AllSignedDataObjects` for all.

#### 5.2.4 DataObjectFormat
- Signed. Describes format of one signed data object. `ObjectReference` **required**. At least one of `Description`, `ObjectIdentifier`, `MimeType` **shall** be present.

#### 5.2.5 SignatureProductionPlaceV2
- Signed. Address of signer (city, street, state, postal code, country). Empty **shall not** be generated.

#### 5.2.6 SignerRoleV2
- Signed. Roles: claimed, certified (X.509 attribute certificate or other), or signed assertions. Empty **shall not** be generated.

#### 5.2.7 Countersignatures
- **CounterSignature** unsigned property containing `ds:Signature`. The countersignature’s `ds:Reference` **shall** reference `ds:SignatureValue` of the embedding signature with `Type="http://uri.etsi.org/01903#CountersignedSignature"`.

#### 5.2.8 Time‑stamps on Signed Data Objects
##### 5.2.8.1 AllDataObjectsTimeStamp
- Signed. Contains electronic time‑stamps on all `ds:Reference` in `ds:SignedInfo` except the one referencing `SignedProperties`. Implicit mechanism.

##### 5.2.8.2 IndividualDataObjectsTimeStamp
- Signed. Explicit `Include` elements for selected `ds:Reference` (excluding `SignedProperties`). `referencedData="true"` required.

#### 5.2.9 SignaturePolicyIdentifier
- Signed. Either `SignaturePolicyId` (with `SigPolicyId`, hash, optional transforms) or `SignaturePolicyImplied`. Qualifiers: `SPURI`, `SPUserNotice`, `SPDocSpecification`.

#### 5.2.10 SignaturePolicyStore
- Unsigned. Contains signature policy document (base‑64) or local URI. **Shall** only be used if `SignaturePolicyIdentifier` contains `SigPolicyHash`.

### 5.3 SignatureTimeStamp
- Unsigned. Implicit mechanism: time‑stamps `ds:SignatureValue`.

### 5.4 Qualifying Properties for Validation Data Values
#### 5.4.1 CertificateValues
- Unsigned. Contains certificates: trust anchor, CA path (if not in `ds:KeyInfo`), signing certificate (if not in `ds:KeyInfo`), signers of revocation data, and countersignature certificates. `EncapsulatedX509Certificate` or `OtherCertificate`.

#### 5.4.2 RevocationValues
- Unsigned. Contains CRLs, OCSP responses, and other revocation data. For signing certificate, CA certificates, and optional for countersignatures.

#### 5.4.3 AttrAuthoritiesCertValues
- Unsigned. Certificates for attribute authorities and signed assertions (signing certificates, CA path, revocation signers).

#### 5.4.4 AttributeRevocationValues
- Unsigned. Revocation values for attribute authority certificates.

### 5.5 Qualifying Properties for Long‑Term Availability and Integrity
#### 5.5.1 TimeStampValidationData
- Unsigned. Contains `CertificateValues` and `RevocationValues` for verifying electronic time‑stamps. `URI` attribute when used for signed time‑stamp containers.

#### 5.5.2 ArchiveTimeStamp (namespace v1.4.1)
- Unsigned. Implicit (not distributed) or explicit (distributed) mechanism. Computes digest over all signature elements and unsigned properties before it.

#### 5.5.3 RenewedDigests
- Unsigned. Contains recomputed digests of detached data objects with a different algorithm when digest algorithm becomes weak.

## 6 XAdES Baseline Signatures
### 6.1 Signature Levels
- B‑B: signed properties and some unsigned.
- B‑T: adds signature time‑stamp.
- B‑LT: adds validation data (certificates, revocation).
- B‑LTA: adds archive time‑stamps for long‑term preservation.

### 6.2.1 Algorithm Requirements
- Shall comply with ETSI TS 119 312. MD5 **shall not** be used.

### 6.2.2 Notation for Requirements

**Additional requirements** (selected):
- ds:KeyInfo/X509Data: signing certificate **shall** be included; generators **should** include other certificates for path building.
- CanonicalizationMethod: one of six allowed URIs; comment‑variant **should not** be used.
- SigningTime: value **shall** be claimed UTC time.
- SigningCertificateV2: `IssuerSerialV2` and `URI` **shall not** be present.
- DataObjectFormat: one per signed data object (except `SignedProperties`).
- SignaturePolicyStore: only if `SignaturePolicyIdentifier` has `SigPolicyHash`.
- SignatureTimeStamp: each element contains exactly one time‑stamp; time‑stamp must be before certificate revocation/expiry.
- CertificateValues and RevocationValues: inclusion determined by steps 4.a/4.b of ArchiveTimeStamp algorithm.
- ArchiveTimeStamp v1.4.1: before generation, all validation material **shall** be included (certificates and status for signer, countersigners, attribute certs, previous time‑stamps).

### 6.4 Legacy XAdES Baseline Signatures
When new unsigned qualifying properties are added to legacy XAdES baseline signatures, they **shall** comply with the present document.

## Annex A (Normative): Additional Qualifying Properties
### A.1.1 CompleteCertificateRefsV2
Unsigned. References to trust anchor and CA certificates (not signing certificate). **Shall** be present in `ds:KeyInfo` or `CertificateValues`.

### A.1.2 CompleteRevocationRefs
Unsigned. References to revocation values for signing certificate and CA certificates. Supports CRLRefs, OCSPRefs, OtherRefs.

### A.1.3 AttributeCertificateRefsV2
Unsigned. References for attribute authority certificates. Not in `CompleteCertificateRefsV2` or `SigningCertificateV2`.

### A.1.4 AttributeRevocationRefs
Unsigned. References for revocation values of attribute authority certificates.

### A.1.5 Time‑stamps on References to Validation Data
#### A.1.5.1 SigAndRefsTimeStampV2
Unsigned. Time‑stamps `ds:SignatureValue`, `SignatureTimeStamp`, and all reference lists (`CompleteCertificateRefsV2`, `CompleteRevocationRefs`, etc.). Implicit or explicit based on distribution.

#### A.1.5.2 RefsOnlyTimeStampV2
Unsigned. Time‑stamps only the reference lists (no signature value).
