# ETSI EN 319 162-2: Electronic Signatures and Infrastructures (ESI); Associated Signature Containers (ASiC); Part 2: Additional ASiC containers
**Source**: ETSI (European Telecommunications Standards Institute) | **Version**: V1.1.1 | **Date**: 2016-04 | **Type**: Normative (European Standard)
**Original**: [ETSI EN 319 162-2 V1.1.1](http://www.etsi.org/standards-search)

## Scope (Summary)
This document defines additional ASiC containers for use cases not covered by the baseline containers specified in ETSI EN 319 162-1 [2]. It specifies ASiC-S Time assertion, ASiC-E CAdES, and ASiC-E Time assertion additional containers, and provides recommendations for using ASiC with existing container specifications (OCF, ODF, UCF) and archival systems using Evidence Records (ER).

## 4 ASiC additional containers
### 4.1 Introduction
This clause defines ASiC additional containers designed to maximize interoperability for ASiC for use cases considered of general use not covered by baseline containers specified in ASiC part 1 [2] clause 5:
- ASiC-S with time assertion specified in clause 4.2.1;
- ASiC-E with CAdES specified in clause 4.3.1; and
- ASiC-E with time assertion specified in clause 4.3.2.

### 4.2 ASiC-S additional containers
#### 4.2.1 ASiC-S Time assertion additional container
The requirement specified in ASiC part 1 [2] for ASiC-S shall apply with the following additional restrictions:
a) it shall comply with the specific provision given in ASiC part 1 [2], clause 4.3.3.1 item 2) a) and clause 4.3.3.2, item 4) a), 4) d) or 4) e);
b) it may contain additional elements in the META-INF folder as specified in ASiC part 1 [2], clause 4.3.3.2 item 5) a) and 5) b);
c) it shall support ASiC part 1 [2], clause 4.3.3.2 item 2); and
d) if one or more ASiCArchiveManifest files are present they shall comply with ASiC part 1 [2], clause A.7 with the additional restriction that only SignedData shall be used to include certificate and revocation information.

### 4.3 ASiC-E additional containers
#### 4.3.1 ASiC-E CAdES additional container
The requirements specified in ASiC part 1 [2] clause 4.4 for ASiC-E shall apply with the following additional restrictions:
a) it shall comply with the specific provision given in ASiC part 1 [2], clause 4.4.4.1, item 1) a), and clause 4.4.4.2, item 3) a); and
b) it shall support ASiC part 1 [2], clause 4.4.5, item 2).
NOTE: Implementations compliant with ASiC-E CAdES additional container requirements can support the same levels defined for baseline containers specified in ASiC part 1 [2], clause 5.

#### 4.3.2 ASiC-E Time assertion additional container
The requirement specified in ASiC part 1 [2] for ASiC-E shall apply with the following additional restrictions:
a) it shall comply with the specific provision given in ASiC part 1 [2], clause 4.4.4.1, item 1) a), and clause 4.4.4.2, item 3) b) or 4) a) or 4) b); and
b) it shall support ASiC part 1 [2], clause 4.4.5 item 2).

## Informative Annexes (Condensed)
- **Annex A (informative) – Using ASiC with existing container specifications**: Specifies recommendations for using ASiC containers compatibly with OCF, ODF, and UCF. The container may be physical (ZIP) or abstract. Recommends presence of "mimetype", "META-INF/container.xml", "META-INF/manifest.xml", "META-INF/metadata.xml", and signature files with specific conformance requirements per format. Table A.1 summarizes requirements.
- **Annex B (informative) – ASiC application to archival systems using ER**: Describes ways to support extraction of signed data objects from archival systems with Evidence Records. For single extraction, ASiC-S container should be created including signed data and ER file named per ASiC part 1 [2] clauses. For multiple extractions, ASiC-E container should be created with each signed data file, associated ERs, and an ASiCEvidenceRecordManifest referencing ER and data objects. The ER hashtree may be reduced per IETF RFC 4998 or RFC 6283.
- **Annex C (informative) – Example of ASiC-E with CAdES and time-stamp token**: Provides a concrete example of an ASiC-E container initially containing two XML files and a CAdES signature, later updated with two PDF files and a time-stamp token. Shows XML content of ASiCManifest1.xml and ASiCManifest2.xml including SigReference and DataObjectReference elements.
